📋 Journalctl — Logs do Sistema

O Detetive do Sistema Linux — Desvende os mistérios dos logs!

🤔 O que é o Journalctl?

Conceito Descrição
Logs Registros automáticos de tudo que acontece no sistema
journalctl Comando para ler, filtrar e pesquisar esses logs

🎯 Analogia: Imagine um diário automático do computador:

  • 📅 "10:30 - Usuário fez login"
  • ⚠️ "10:35 - Apache não iniciou"
  • ✅ "10:40 - Backup concluído"
exemplo de log em tempo real
usuario@linux:~$ journalctl -f
Jan 15 10:30:45 systemd[1]: Started Daily apt activities.
Jan 15 10:30:46 sshd[1234]: Accepted publickey for usuario
█ ← Esperando novos logs...

💼 Por que é importante: Quando algo dá errado, os logs são sua única pista!

🎉 Seus Primeiros Comandos

comandos básicos
# Ver todos os logs (muito texto!)
usuario@linux:~$ journalctl

# Ver logs em TEMPO REAL (o mais usado!)
usuario@linux:~$ journalctl -f
Jan 15 10:30:45 systemd[1]: Starting Daily apt...
Jan 15 10:30:46 sshd[1234]: Accepted publickey for usuario
█ ← Esperando novos logs...

💡 Analogia do -f: É como assistir uma câmera de segurança ao vivo!

🔧 Filtrar por Serviço: -u

Ver logs de um serviço específico!

filtrar por serviço
# Logs do SSH
usuario@linux:~$ journalctl -u ssh
Jan 15 10:00:00 sshd[1000]: Server listening on port 22
Jan 15 10:30:46 sshd[1234]: Accepted publickey for usuario

# Logs do Apache
usuario@linux:~$ journalctl -u apache2

# Logs do Nginx em tempo real
usuario@linux:~$ journalctl -u nginx -f

💡 Como saber o nome do serviço? Use systemctl list-units --type=service

⏰ Filtrar por Tempo

filtros de tempo
# Logs de hoje
usuario@linux:~$ journalctl --since today

# Logs de ontem
usuario@linux:~$ journalctl --since yesterday

# Logs da última hora
usuario@linux:~$ journalctl --since "1 hour ago"

# Logs entre duas datas
usuario@linux:~$ journalctl --since "2024-01-10" --until "2024-01-15"

# Últimas 50 linhas
usuario@linux:~$ journalctl -n 50

💡 Caso de uso: "O servidor parou às 3h, o que aconteceu?"
→ journalctl --since "03:00" --until "03:30"

🚨 Filtrar por Prioridade: -p

Nem todo log é igual. Alguns são mais urgentes!

Nível Nome Uso
err Erros Problemas que precisam de atenção imediata
warning Avisos Alertas que podem virar problemas
info Informações Operações normais do sistema
filtrar por prioridade
# Ver apenas ERROS (o mais útil!)
usuario@linux:~$ journalctl -p err

# Ver WARNINGS e erros
usuario@linux:~$ journalctl -p warning

# Erros de um serviço específico
usuario@linux:~$ journalctl -u apache2 -p err

✅ Dica de ouro: Quando algo dá errado, comece SEMPRE por journalctl -p err!

🔄 Filtrar por Boot: -b

filtros de boot
# Logs do boot atual
usuario@linux:~$ journalctl -b

# Logs do boot anterior (antes de reiniciar)
usuario@linux:~$ journalctl -b -1

# Listar todos os boots
usuario@linux:~$ journalctl --list-boots
-2 abc123... Mon 2024-01-13 08:00 - Mon 2024-01-13 23:59
-1 def456... Tue 2024-01-14 08:00 - Tue 2024-01-14 23:59
 0 ghi789... Wed 2024-01-15 08:00 - Wed 2024-01-15 10:30

💡 Quando usar? "O servidor reiniciou sozinho. O que aconteceu?" → journalctl -b -1

🖥️ Kernel e Combinações

logs do kernel e combinações
# Logs do kernel (problemas de hardware!)
usuario@linux:~$ journalctl -k
Jan 15 08:00:01 kernel: Linux version 5.15.0
Jan 15 08:00:03 kernel: eth0: Link is Up - 1Gbps

# Só erros do kernel
usuario@linux:~$ journalctl -k -p err

# Combinar: erros do SSH hoje
usuario@linux:~$ journalctl -u ssh -p err --since today

# Buscar palavra específica
usuario@linux:~$ journalctl | grep "Failed"

💡 Lógica de investigação:
Serviço (-u) + Prioridade (-p) + Tempo (--since) + Busca (grep)

🔎 Caso Prático: SSH não Funciona

⚠️ 😱 O SSH parou de funcionar! Como investigar?

investigação passo a passo
# 1. Ver status do serviço
usuario@linux:~$ systemctl status ssh
● ssh.service - OpenBSD Secure Shell server
   Active: failed (Result: exit-code)

# 2. Ver logs recentes do SSH
usuario@linux:~$ journalctl -u ssh -n 30

# 3. Filtrar só erros
usuario@linux:~$ journalctl -u ssh -p err
Jan 15 10:30:00 sshd: error: Bind to port 22 failed: Address already in use

# 4. AHA! Porta 22 em uso!
usuario@linux:~$ sudo lsof -i :22

💾 Gerenciando Espaço

⚠️ Logs podem ocupar MUITO espaço!

gerenciamento de espaço dos logs
# Ver espaço ocupado
usuario@linux:~$ journalctl --disk-usage
Archived and active journals take up 1.2G

# Limpar (manter só 500MB)
root@linux:~# journalctl --vacuum-size=500M

# Limpar logs > 7 dias
root@linux:~# journalctl --vacuum-time=7d

⚙️ Configuração permanente: Edite /etc/systemd/journald.conf e defina SystemMaxUse=500M

❓ Verifique seu Conhecimento

O que é o journald no contexto do systemd?

AUm servidor de logs remoto
BO serviço de coleta e armazenamento de logs do systemd
CUm editor de arquivos de log
DUm analisador de segurança de logs
✓ O systemd-journald coleta logs de serviços, kernel e aplicações, armazenando em formato binário. O journalctl é a ferramenta para consultá-los.

Qual comando exibe os logs de um serviço específico em tempo real?

Ajournalctl --service=nginx --live
Bjournalctl -u nginx --monitor
Cjournalctl -u nginx -f
Djournalctl --follow nginx.service
✓ -u filtra por unidade (serviço) e -f segue em tempo real (como tail -f). Ex: journalctl -u apache2 -f.

Como filtrar logs do journalctl por prioridade (apenas erros e críticos)?

Ajournalctl -p err
Bjournalctl --level=error
Cjournalctl -e critical
Djournalctl --filter=err
✓ -p filtra por prioridade: emerg, alert, crit, err, warning, notice, info, debug. journalctl -p err mostra err e mais graves.

Qual comando limita a saída do journalctl às últimas 50 linhas?

Ajournalctl --last=50
Bjournalctl --limit 50
Cjournalctl --tail=50
Djournalctl -n 50
✓ -n N mostra as N linhas mais recentes (padrão 10). Combine com -f: journalctl -n 50 -f para ver últimas 50 e continuar monitorando.
