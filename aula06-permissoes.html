🔐 Permissões de Arquivos

Controle quem acessa o quê (rwx) — Comandos chmod, chown e grupos!

🤔 Por que Permissões Existem?

Segurança = Controle de Acesso

Analogia: Escritório No Linux
Nem todo mundo pode entrar em todas as salas Cada arquivo tem um "dono"
Documentos confidenciais ficam trancados Usuários pertencem a "grupos"
Cada funcionário tem seu crachá 3 tipos de acesso: ler, escrever, executar
Gerentes têm mais acesso que estagiários Root pode tudo (é o "chefão")

⚠️ Sem permissões...

  • Qualquer usuário poderia apagar arquivos do sistema
  • Dados pessoais ficariam expostos
  • Vírus teriam acesso total ao computador
  • Seria impossível ter múltiplos usuários com segurança
exemplo de permissões
usuario@linux:~$ ls -l arquivo.txt
-rwxr-xr-- 1 usuario grupo 1234 Jan 15 arquivo.txt
# Quem pode ler, escrever e executar? 🤔

📝 Os 3 Tipos de Permissão: RWX

Símbolo Nome Valor Octal Em Arquivo Em Diretório
r 📖 Read (leitura) 4 Ver o conteúdo Listar arquivos (ls)
w ✏️ Write (escrita) 2 Modificar o arquivo Criar/deletar arquivos dentro
x ⚡ Execute (execução) 1 Rodar como programa Entrar no diretório (cd)
- Sem permissão 0 Nenhum acesso Nenhum acesso

👥 Os 3 Níveis de Usuário

Cada arquivo define permissões para 3 categorias de usuários:

Símbolo Categoria Descrição Posição
u 👤 Owner (dono) Quem criou o arquivo 1º bloco rwx
g 👥 Group (grupo) Membros do grupo do arquivo 2º bloco rwx
o 🌍 Others (outros) Todos os demais usuários 3º bloco rwx

Leitura de uma string de permissões: -rwxr-xr--

  • - — Tipo (arquivo, d=diretório, l=link)
  • rwx — Permissões do dono
  • r-x — Permissões do grupo
  • r-- — Permissões dos outros
ls -l — lendo permissões
usuario@linux:~$ ls -l
drwxr-xr-x  2 usuario grupo 4096 Jan 15 Documentos
-rw-r--r--  1 usuario grupo 1234 Jan 15 arquivo.txt
-rwxr-x---  1 usuario grupo 5678 Jan 15 script.sh
lrwxrwxrwx  1 usuario grupo   12 Jan 15 link -> arquivo.txt
Posição Significado Valores
1º caractereTipo do arquivo-=arquivo, d=diretório, l=link
2º–4ºPermissões do DONOrwx ou ---
5º–7ºPermissões do GRUPOrwx ou ---
8º–10ºPermissões de OUTROSrwx ou ---

💡 O traço (-) significa "sem permissão": r-- = só leitura | rw- = leitura e escrita | rwx = acesso total

🔢 Sistema Octal (Numérico)

Cada permissão tem um valor numérico. Somando os valores, obtém-se o número octal da permissão:

Combinação Cálculo Octal
rwx4+2+17
rw-4+2+06
r-x4+0+15
r--4+0+04
-wx0+2+13
-w-0+2+02
--x0+0+11
---0+0+00
Octal Significado Uso Típico
755rwxr-xr-xDiretórios e scripts executáveis
644rw-r--r--Arquivos comuns, configurações
700rwx------Só o dono acessa
600rw-------Arquivo privado (ex: chaves SSH)
750rwxr-x---Dono total, grupo lê/executa
777rwxrwxrwx⚠️ Todos podem tudo (perigoso!)

💡 Memorize: 755 e 644
755 = rwxr-xr-x (diretórios e scripts)
644 = rw-r--r-- (documentos e configs)

🔧 chmod — Modo Simbólico

chmod = Change Mode (mudar permissões)

Sintaxe: chmod [quem][operador][permissão] arquivo

Quem Operador Permissão
u = user (dono)
g = group
o = others
a = all (todos) 		+ = adicionar
- = remover
= = definir exato 		r = read
w = write
x = execute
chmod modo simbólico
# Adicionar execução para o dono
usuario@linux:~$ chmod u+x script.sh

# Remover escrita do grupo e outros
usuario@linux:~$ chmod go-w documento.txt

# Definir permissão exata para todos
usuario@linux:~$ chmod a=r arquivo.txt

# Múltiplas mudanças de uma vez
usuario@linux:~$ chmod u+rwx,g+rx,o-rwx programa

🔧 chmod — Modo Octal

Modo octal é mais rápido e direto!

chmod modo octal
# Sintaxe: chmod NNN arquivo
# Onde NNN = dono-grupo-outros

# Permissão padrão para arquivo (rw-r--r--)
usuario@linux:~$ chmod 644 arquivo.txt

# Permissão para script (rwxr-xr-x)
usuario@linux:~$ chmod 755 script.sh

# Só o dono acessa (rwx------)
usuario@linux:~$ chmod 700 pasta_privada

# Arquivo secreto (rw-------)
usuario@linux:~$ chmod 600 chave_ssh
Vantagens do Octal Vantagens do Simbólico
Define TUDO de uma vez Mais intuitivo para iniciantes
Mais curto de digitar Modifica só o necessário
Padrão em scripts Mais legível em scripts

📁 chmod Recursivo (-R)

Aplicar permissões em toda uma pasta e seu conteúdo:

chmod -R
# Aplicar 755 em tudo dentro de /var/www
usuario@linux:~$ chmod -R 755 /var/www

# Adicionar leitura para todos, recursivamente
usuario@linux:~$ chmod -R a+r documentos/

# 755 só para diretórios
find pasta/ -type d -exec chmod 755 {} \;

# 644 só para arquivos
find pasta/ -type f -exec chmod 644 {} \;

⚠️ Cuidado com -R!

  • Não use em diretórios do sistema (/, /etc, /bin)
  • Verifique DUAS VEZES antes de executar
  • chmod -R 777 é quase sempre ERRADO e perigoso!

👤 chown — Mudar Dono e Grupo

chown = Change Owner (mudar proprietário)

chown — exemplos
# Mudar dono do arquivo
root@linux:~# chown maria arquivo.txt

# Mudar dono e grupo
root@linux:~# chown maria:desenvolvedores arquivo.txt

# Mudar só o grupo (duas formas)
root@linux:~# chown :webteam site/
root@linux:~# chgrp webteam site/

# Recursivo (-R)
root@linux:~# chown -R www-data:www-data /var/www/html
Sintaxe Ação
chown usuario arquivoMuda apenas o dono
chown usuario:grupo arquivoMuda dono e grupo
chown :grupo arquivoMuda apenas o grupo
chgrp grupo arquivoMuda apenas o grupo (alternativo)

⚠️ Só root pode mudar o dono de arquivos! Usuário comum só pode mudar o grupo para grupos dos quais faz parte.

👥 Trabalhando com Grupos

Grupos permitem compartilhar acesso entre usuários!

consultando grupos
# Ver grupos do usuário atual
usuario@linux:~$ groups
usuario sudo docker www-data

# Ver grupos de outro usuário
usuario@linux:~$ groups maria
maria : maria desenvolvedores

# Ver detalhes com id
usuario@linux:~$ id
uid=1000(usuario) gid=1000(usuario) groups=1000(usuario),27(sudo),999(docker)

# Listar todos os grupos do sistema
usuario@linux:~$ cat /etc/group
gerenciando grupos (como root)
# Criar novo grupo
root:~# groupadd desenvolvedores

# Adicionar usuário ao grupo
root:~# usermod -aG desenvolvedores maria

# Remover usuário do grupo
root:~# gpasswd -d maria desenvolvedores

🎭 umask — Permissões Padrão

umask define as permissões padrão para novos arquivos!

umask é uma máscara de subtração: Permissão Base − umask = Permissão Final

  • Base para arquivos: 666 (rw-rw-rw-)
  • Base para diretórios: 777 (rwxrwxrwx)
umask Arquivos resultantes Diretórios resultantes
022644 (rw-r--r--)755 (rwxr-xr-x)
027640 (rw-r-----)750 (rwxr-x---)
077600 (rw-------)700 (rwx------)
usando umask
# Ver umask atual
usuario@linux:~$ umask
0022

# Ver em formato simbólico
usuario@linux:~$ umask -S
u=rwx,g=rx,o=rx

# Mudar umask temporariamente
usuario@linux:~$ umask 077
# Arquivos: 600, Diretórios: 700

⚡ Permissões Especiais: SUID

⚠️ SUID = Set User ID — Executar como dono

Quando um arquivo tem SUID, ele executa com as permissões do dono, não do usuário que o executou.

Exemplo clássico: o comando passwd — usuários comuns podem mudar sua própria senha, mas o arquivo /etc/shadow só root pode editar!

Aparece um s no lugar do x do dono: -rwsr-xr-x — Valor octal: 4xxx

SUID na prática
# Ver arquivos SUID no sistema
usuario@linux:~$ ls -l /usr/bin/passwd
-rwsr-xr-x 1 root root 68208 Jan 15 /usr/bin/passwd

# Adicionar SUID
root@linux:~# chmod u+s programa
root@linux:~# chmod 4755 programa

⚠️ SUID é um risco de segurança se mal usado. Nunca coloque em scripts!

⚡ Permissões Especiais: SGID e Sticky Bit

SGID = Set Group ID — Valor octal: 2xxx

Em diretórios: arquivos criados dentro herdam o grupo do diretório, não do usuário. Útil para pastas compartilhadas entre equipes!

SGID em diretório compartilhado
root@linux:~# mkdir /projetos/devs
root@linux:~# chgrp desenvolvedores /projetos/devs
root@linux:~# chmod 2775 /projetos/devs

# Agora qualquer arquivo criado terá grupo "desenvolvedores"!
maria@linux:/projetos/devs$ touch codigo.py
maria@linux:/projetos/devs$ ls -l codigo.py
-rw-rw-r-- 1 maria desenvolvedores 0 Jan 15 codigo.py

Sticky Bit = Proteção contra deleção — Valor octal: 1xxx

Em diretórios com Sticky Bit, usuários só podem deletar seus próprios arquivos, mesmo tendo permissão de escrita no diretório. Exemplo clássico: o diretório /tmp

sticky bit
# Ver sticky bit no /tmp
usuario@linux:~$ ls -ld /tmp
drwxrwxrwt 15 root root 4096 Jan 15 /tmp
# O "t" no final indica sticky bit!

# Adicionar sticky bit
root@linux:~# chmod +t /pasta_compartilhada
root@linux:~# chmod 1777 /pasta_compartilhada
Octal Nome Efeito
4xxxSUIDExecuta como dono
2xxxSGIDHerda grupo do diretório
1xxxSticky BitProteção de deleção

🌐 Caso Prático: Servidor Web

Configurando permissões corretas para /var/www/html:

configuração do servidor web
# 1. Definir dono como www-data (usuário do Apache/Nginx)
root@linux:~# chown -R www-data:www-data /var/www/html

# 2. Permissões para diretórios (755)
root@linux:~# find /var/www/html -type d -exec chmod 755 {} \;

# 3. Permissões para arquivos (644)
root@linux:~# find /var/www/html -type f -exec chmod 644 {} \;

# 4. Adicionar seu usuário ao grupo www-data
root@linux:~# usermod -aG www-data seu_usuario

# 5. Permitir que o grupo escreva
root@linux:~# chmod -R g+w /var/www/html

💡 Por que essas permissões?
755 dirs: Web server pode entrar e ler
644 files: Web server pode ler, mas não modificar
www-data: Usuário padrão do Apache/Nginx

🔑 Caso Prático: Chaves SSH

⚠️ SSH é MUITO exigente com permissões!

permissões obrigatórias para SSH
# Diretório ~/.ssh (700 - só o dono)
usuario@linux:~$ chmod 700 ~/.ssh

# Chave privada (600 - só o dono lê/escreve)
usuario@linux:~$ chmod 600 ~/.ssh/id_rsa

# Chave pública (644 - todos podem ler)
usuario@linux:~$ chmod 644 ~/.ssh/id_rsa.pub

# Arquivo authorized_keys (600)
usuario@linux:~$ chmod 600 ~/.ssh/authorized_keys

# Arquivo config (600)
usuario@linux:~$ chmod 600 ~/.ssh/config

⚠️ Se as permissões estiverem erradas:
Permissions 0644 for '/home/user/.ssh/id_rsa' are too open.
It is required that your private key files are NOT accessible by others.

❓ Verifique seu Conhecimento

O que representa a saída -rwxr-xr-- nas permissões de um arquivo?

ADiretório com permissão total para todos
BArquivo: dono=rwx, grupo=r-x, outros=r--
CLink simbólico com permissão de leitura
DArquivo especial com permissões restritas
✓ O primeiro caractere indica o tipo (-=arquivo, d=diretório, l=link). Os 9 seguintes são 3 grupos de rwx: dono, grupo, outros.

Qual comando define permissão 755 para o arquivo script.sh?

Achown 755 script.sh
Bsetperm 755 script.sh
Cchmod 755 script.sh
Dchgrp 755 script.sh
✓ chmod altera permissões. 755 = dono(rwx=7) grupo(r-x=5) outros(r-x=5). Use notação simbólica: chmod u+x,go-w arquivo.

Qual comando muda o dono e o grupo de um arquivo para ‘www-data’?

Achown www-data:www-data arquivo
Bchmod www-data arquivo
Cchgrp -u www-data arquivo
Dusermod --file www-data arquivo
✓ chown dono:grupo arquivo altera ambos. chown dono arquivo altera só o dono. chgrp grupo arquivo altera só o grupo.

O que representa o valor octal 4 em permissões de arquivo?

APermissão de execução (x)
BPermissão de escrita (w)
CSem permissão (---)
DPermissão de leitura (r)
✓ r=4, w=2, x=1. Some para combinar: rwx=7, rw-=6, r-x=5, r--=4, -wx=3, -w-=2, --x=1, ---=0.
