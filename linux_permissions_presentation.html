🏠 Analogia: Arquivos como Gavetas de Casa

🤔 Vamos imaginar cada arquivo como uma gaveta...

E cada permissão como um tipo diferente de chave!

📂 Arquivo = Gaveta/Armário

Cada gaveta tem etiquetas (nomes)
Tem coisas guardadas dentro (conteúdo)
Algumas são gavetas especiais (executáveis)
Cada uma tem um dono principal

🔑 r = Chave da "Leitura"

Permite ver a etiqueta da gaveta
e saber o que tem dentro.
Como espreitar pela fechadura! 👀

👥 Três grupos de pessoas

Dono: quem comprou a gaveta
Grupo: família/colegas de quarto
Outros: qualquer visita

✏️ w = Chave da "Escrita"

Permite mexer no conteúdo,
guardar e tirar coisas da gaveta.
É o poder de modificar! ✍️

⚡ x = Chave da "Execução"

Para gavetas especiais (ferramentas),
permite usar como ferramenta.
Para entrar em cômodos! 🚪

user@casa:~$ ls -la gaveta_secreta.txt
-rw------- 1 joao familia 50 jun 29 gaveta_secreta.txt
# João pode ler e escrever, família e outros: nem ver!

🔍 Decifrando o Código rwx

🕵️ Vamos quebrar esse código misterioso

Cada letra tem um significado específico!

📊 Anatomia das permissões

user@linux:~$ ls -la exemplo.txt
-rw-r--r-- 1 maria devs 1024 jun 29 exemplo.txt
-rw-r--r--
│││ │││ │││
│││ │││ └┴┴─ Outros: r-- (só ler)
│││ └┴┴───── Grupo: r-- (só ler)
│└┴─────── Dono: rw- (ler + escrever)
└─────────── Tipo: - (arquivo comum)

r=read (ler) w=write (escrever) x=execute (executar)

📄 Para Arquivos

r - Ler o conteúdo
w - Modificar o conteúdo
x - Executar como programa

Exemplo: Um script com rwx pode ser lido, editado e executado!

📁 Para Diretórios

r - Listar arquivos (ls)
w - Criar/remover arquivos
x - Entrar no diretório (cd)

Importante: Sem x, não consegue nem entrar na pasta!

⚠️ Pegadinha dos Diretórios!

Para acessar um arquivo dentro de uma pasta, você precisa de permissão x na pasta!
É como precisar da chave do cômodo para chegar até a gaveta.

🎯 Exemplos Práticos

user@linux:~$ ls -la
drwxr-xr-x 2 user group 4096 jun 29 pasta_publica/
-rwx------ 1 user group 256 jun 29 script_pessoal.sh
-rw-r--r-- 1 user group 1024 jun 29 documento.txt
# pasta_publica: todos podem entrar e listar
# script_pessoal: só o dono pode tudo
# documento: dono escreve, outros só leem
Slide 4 de 11

🔢 chmod: O Poder dos Números

🧮 Sistema octal: A calculadora das permissões

Cada permissão tem um valor, e somamos para formar o número final!

⚡ A Matemática Mágica

r = 4 w = 2 x = 1

rwx = 4 + 2 + 1 = 7 (pode tudo)
rw- = 4 + 2 + 0 = 6 (ler e escrever)
r-x = 4 + 0 + 1 = 5 (ler e executar)
r-- = 4 + 0 + 0 = 4 (só ler)
--- = 0 + 0 + 0 = 0 (nada!)

📋 Permissões Mais Usadas

NúmeroPermissãoUso
755rwxr-xr-xScripts/programas
644rw-r--r--Arquivos de texto
600rw-------Arquivos privados
700rwx------Pasta privada
777rwxrwxrwx⚠️ Perigoso! Todos podem tudo

⚡ Usando chmod

user@linux:~$ chmod 755 script.sh
# Script executável para todos
user@linux:~$ chmod 644 documento.txt
# Arquivo de leitura geral
user@linux:~$ chmod 600 senha.txt
# Arquivo ultra secreto
user@linux:~$ chmod -R 755 pasta/
# Recursivo: aplica a pasta toda

🎯 Exemplo Passo a Passo

user@linux:~$ touch exemplo.txt
# Cria arquivo
user@linux:~$ ls -la exemplo.txt
-rw-r--r-- 1 user group 0 jun 29 exemplo.txt
user@linux:~$ chmod 700 exemplo.txt
# Agora só eu posso acessar
user@linux:~$ ls -la exemplo.txt
-rwx------ 1 user group 0 jun 29 exemplo.txt

💀 CUIDADO com 777!

chmod 777 dá permissão total para TODOS!
É como deixar sua casa aberta com um cartaz "pode entrar".
Use apenas para testes, nunca em produção!

💡 Dica para Memorizar

7 = Tudo (rwx) - Super usuário
5 = Ler e executar (r-x) - Visitante educado
4 = Só ler (r--) - Curioso respeitoso
0 = Nada (---) - Porta trancada

Slide 5 de 11

🎭 chmod: Modo Simbólico

🎨 Mais intuitivo e flexível

Ao invés de números, use símbolos para adicionar/remover permissões específicas!

🧩 Os Três Componentes

QUEM + OPERAÇÃO + PERMISSÃO

u = user (dono) + = adicionar r = read
g = group (grupo) - = remover w = write
o = others (outros) = = definir x = execute
a = all (todos)

➕ Adicionando Permissões

user@linux:~$ chmod +x script.sh
# Todos podem executar
user@linux:~$ chmod u+w arquivo.txt
# Dono pode escrever
user@linux:~$ chmod g+r,o+r documento.txt
# Grupo e outros podem ler
user@linux:~$ chmod a+x ferramenta
# Todos podem executar

➖ Removendo Permissões

user@linux:~$ chmod -w arquivo.txt
# Ninguém pode escrever
user@linux:~$ chmod o-r secreto.txt
# Outros não podem ler
user@linux:~$ chmod g-w,o-w pasta/
# Grupo e outros: sem escrita
user@linux:~$ chmod a-x programa
# Ninguém executa

⚡ Exemplos Práticos

user@linux:~$ ls -la script.sh
-rw-r--r-- 1 user group 256 jun 29 script.sh
user@linux:~$ chmod u+x script.sh
# Dou permissão de execução para mim
user@linux:~$ ls -la script.sh
-rwxr--r-- 1 user group 256 jun 29 script.sh
user@linux:~$ chmod go+x script.sh
# Grupo e outros também podem executar
user@linux:~$ ls -la script.sh
-rwxr-xr-x 1 user group 256 jun 29 script.sh

💡 Quando usar cada modo?

Numérico (755): Quando você sabe exatamente a permissão final
Simbólico (+x): Quando quer fazer ajustes específicos
Recursivo (-R): Para aplicar em pastas inteiras

🏋️ Exercício Rápido

user@linux:~$ touch teste.txt # Criar arquivo
user@linux:~$ chmod go-r teste.txt # Só eu posso ler
user@linux:~$ chmod u+x teste.txt # Eu posso executar
user@linux:~$ chmod g+r teste.txt # Grupo pode ler
user@linux:~$ ls -la teste.txt # Ver resultado

👤 chown e chgrp: Mudando o Dono

🔑 Transferindo a posse dos arquivos

Como passar a escritura de uma casa para outra pessoa!

👤 chown - Mudar dono

root@linux:~# chown maria arquivo.txt
# Maria vira dona do arquivo
root@linux:~# chown maria:developers pasta/
# Maria=dono, developers=grupo
root@linux:~# chown -R joao:users projeto/
# Mudança recursiva da pasta
root@linux:~# chown :admin arquivo.txt
# Só muda o grupo para admin

👥 chgrp - Mudar grupo

user@linux:~$ chgrp developers script.sh
# Arquivo pertence ao grupo developers
user@linux:~$ chgrp -R admins pasta/
# Toda pasta vira do grupo admins
user@linux:~$ ls -la script.sh
-rwxr-xr-x 1 user developers 256 jun 29 script.sh

🎯 Exemplo Prático: Transferindo Projeto

root@linux:~# ls -la projeto.zip
-rw-r--r-- 1 root root 2048 jun 29 projeto.zip
root@linux:~# chown maria:developers projeto.zip
# Transferindo para Maria do time developers
root@linux:~# ls -la projeto.zip
-rw-r--r-- 1 maria developers 2048 jun 29 projeto.zip
root@linux:~# chmod g+w projeto.zip
# Grupo developers pode editar também
root@linux:~# ls -la projeto.zip
-rw-rw-r-- 1 maria developers 2048 jun 29 projeto.zip

⚠️ Importante sobre chown

Apenas root pode mudar o dono de arquivos!
Usuários normais só podem mudar o grupo (se fizerem parte dele).
É uma questão de segurança do sistema!

📋 Sintaxes do chown

ComandoResultado
chown user arquivoMuda só o dono
chown user:grupo arquivoMuda dono e grupo
chown :grupo arquivoMuda só o grupo
chown -R user pasta/Recursivo

🔍 Verificando mudanças

user@linux:~$ stat arquivo.txt
File: 'arquivo.txt' Size: 1024 Access: (0644/-rw-r--r--) Uid: (1001/ maria) Gid: (1002/developers)
user@linux:~$ id maria
uid=1001(maria) gid=1001(maria) groups=1001(maria),1002(developers)

💡 Cenários Comuns

Site web: chown www-data:www-data site/
Backup: chown backup:backup dados/
Projeto de equipe: chown :developers projeto/

⭐ Permissões Especiais

🎩 As permissões ninjas do Linux

Para situações especiais que precisam de superpoderes!

🔒 Sticky Bit (t)

root@linux:~# chmod +t /tmp/compartilhado/
# Só o dono pode deletar seus arquivos
root@linux:~# ls -ld /tmp/compartilhado/
drwxrwxrwt 2 root root 4096 jun 29 /tmp/compartilhado/
# Note o 't' no final!

Uso: Pastas públicas onde todos podem criar arquivos, mas só podem deletar os próprios!

👑 SetUID (s)

root@linux:~# chmod u+s programa
# Executa como o dono do arquivo
root@linux:~# ls -la programa
-rwsr-xr-x 1 root root 1024 jun 29 programa
# Note o 's' na posição do 'x'!

⚠️ PERIGO: Muito poder! Use com extremo cuidado!

🎯 Exemplo Prático: Pasta de Projeto

root@linux:~# mkdir /projetos/equipe
root@linux:~# chgrp developers /projetos/equipe
root@linux:~# chmod 2775 /projetos/equipe
# 2 = SetGID, 775 = rwxrwxr-x
root@linux:~# ls -ld /projetos/equipe
drwxrwsr-x 2 root developers 4096 jun 29 /projetos/equipe
# Agora todo arquivo criado aqui vira do grupo developers!

🧮 Números das Permissões Especiais

NúmeroNomeSímboloO que faz
4000SetUIDs (user)Executa como dono do arquivo
2000SetGIDs (group)Arquivos herdam grupo da pasta
1000StickytSó dono pode deletar

💡 Casos de Uso

Sticky: /tmp, pastas compartilhadas
SetGID: Projetos de equipe
SetUID: Programas que precisam de root

🔍 Como usar

chmod 1755 pasta/ # Sticky
chmod 2755 pasta/ # SetGID
chmod 4755 programa # SetUID

⚠️ Cuidados

SetUID pode ser perigoso
Sempre teste antes
Documente o uso

🚨 Segurança

Permissões especiais podem criar brechas de segurança!
Use apenas quando necessário e sempre com conhecimento.

🏋️ Exemplo Prático Completo

🎯 Cenário: Servidor Web da Empresa

Vamos configurar permissões para um site corporativo!

📋 O que precisamos fazer:

Pasta do site: /var/www/empresa
Dono: www-data (servidor web)
Grupo: webdev (desenvolvedores)
Logs apenas para admin
Scripts executáveis, arquivos só leitura

1️⃣ Criando estrutura

root@linux:~# mkdir -p /var/www/empresa/{html,logs,scripts}
root@linux:~# groupadd webdev
root@linux:~# usermod -aG webdev maria
root@linux:~# usermod -aG webdev joao

2️⃣ Definindo donos

root@linux:~# chown -R www-data:webdev /var/www/empresa
root@linux:~# chown -R root:root /var/www/empresa/logs

3️⃣ Configurando permissões

root@linux:~# chmod -R 755 /var/www/empresa
root@linux:~# chmod -R 644 /var/www/empresa/html/*
root@linux:~# chmod -R 755 /var/www/empresa/scripts
root@linux:~# chmod -R 700 /var/www/empresa/logs

4️⃣ SetGID para colaboração

root@linux:~# chmod g+s /var/www/empresa/html
# Novos arquivos herdam grupo webdev

✅ Verificando o resultado

root@linux:~# ls -la /var/www/empresa/
drwxr-xr-x 5 www-data webdev 4096 jun 29 .
drwxrwsr-x 2 www-data webdev 4096 jun 29 html
drwx------ 2 root root 4096 jun 29 logs
drwxr-xr-x 2 www-data webdev 4096 jun 29 scripts
root@linux:~# su - maria
maria@linux:~$ touch /var/www/empresa/html/index.html
# ✅ Maria pode criar arquivos!
maria@linux:~$ ls -la /var/www/empresa/html/index.html
-rw-r--r-- 1 maria webdev 0 jun 29 index.html
# ✅ Arquivo automaticamente do grupo webdev!

🎉 Resultado Final

✅ Servidor web pode ler todos os arquivos
✅ Desenvolvedores podem colaborar na pasta html
✅ Scripts executáveis funcionam corretamente
✅ Logs protegidos, apenas admin acessa
✅ Novos arquivos automaticamente com grupo correto
