🏢 Usuários e Grupos no Linux

O Linux é um sistema multiusuário por natureza. Cada pessoa ou serviço que acessa o sistema possui uma conta com permissões específicas. Pense em uma empresa: cada funcionário tem seu crachá (login), sua mesa (pasta home) e acesso apenas às salas autorizadas (permissões).

👑
Root (UID 0) O superusuário. Pode tudo no sistema. Equivale à chave mestra do prédio. ⚠️ Use com extremo cuidado!
👨‍💻
Usuários Comuns Pessoas reais. Cada uma tem sua pasta home e permissões limitadas. UID: 1000 em diante
⚙️
Usuários de Sistema Contas para serviços (apache, mysql...). Não são pessoas, são programas! UID: 1 a 999

Por que grupos existem?

Grupos permitem gerenciar permissões em massa. Em vez de dar acesso a uma pasta para cada usuário individualmente, você cria um grupo "equipe-dev", adiciona os membros e define permissões uma única vez. Todo usuário tem um grupo primário (criado com o usuário) e pode pertencer a múltiplos grupos secundários.

bash — verificando identidade
usuario@servidor:~$ whoami usuario usuario@servidor:~$ id uid=1000(usuario) gid=1000(usuario) grupos=1000(usuario),27(sudo),100(users) usuario@servidor:~$ groups usuario sudo users

➕ Criando e Gerenciando Usuários

O Linux oferece dois conjuntos de ferramentas para criar usuários: useradd (baixo nível, ideal para scripts) e adduser (interativo, mais amigável em Debian/Ubuntu). Ambos produzem o mesmo resultado, mas com experiências diferentes.

useradd — para scripts

  • Não é interativo
  • Precisa de opções explícitas
  • Disponível em todas as distros
  • Ideal para automação

adduser — para uso manual

  • Interativo e amigável
  • Configura tudo automaticamente
  • Disponível no Debian/Ubuntu
  • Ideal para o dia a dia
bash — criando usuários
# useradd: criar com opções (recomendado para scripts) root@servidor:~# useradd -m -s /bin/bash -c "João Silva" joao root@servidor:~# passwd joao Nova senha: passwd: senha atualizada com sucesso # adduser: interativo, faz tudo (recomendado no dia a dia) root@servidor:~# adduser maria Adicionando usuário 'maria' ... Criando diretório pessoal '/home/maria' ... Nova senha: A informação está correta? [S/n] S

Principais opções do useradd

  • -m — cria a pasta home do usuário (sem isso, não cria!)
  • -s /bin/bash — define o shell padrão do usuário
  • -c "Nome Completo" — comentário com o nome real
  • -g grupo — define o grupo primário
  • -G grupo1,grupo2 — define grupos secundários na criação

⚠️ Atenção! useradd joao sem a opção -m NÃO cria a pasta home. O usuário existirá no sistema, mas sem seu diretório pessoal. Sempre use -m ao criar usuários comuns, ou prefira adduser.

📄 Os Arquivos /etc/passwd e /etc/shadow

Todos os usuários do sistema ficam registrados em /etc/passwd. Este arquivo é legível por todos, pois muitos programas precisam consultar informações de usuários. Por isso, as senhas ficam separadas em /etc/shadow, acessível apenas pelo root.

Estrutura do /etc/passwd

Cada linha representa um usuário, com sete campos separados por ::

usuario : x : 1000 : 1000 : Usuário da Silva : /home/usuario : /bin/bash

Campos do /etc/passwd

  • Login — nome de usuário
  • x — senha está em /etc/shadow
  • UID — identificador numérico único
  • GID — grupo primário
  • Info — nome completo ou comentário
  • Home — diretório pessoal
  • Shell — shell padrão do usuário

/etc/shadow — segurança de senhas

  • Apenas root pode ler este arquivo
  • A senha é armazenada como hash — impossível de reverter
  • $6$ no hash indica SHA-512 (muito seguro)
  • Contém políticas de expiração de senha
  • Tentar ler sem permissão retorna: Permissão negada
bash — consultando usuários
usuario@servidor:~$ cat /etc/passwd | grep "joao" joao:x:1001:1001:João Silva:/home/joao:/bin/bash usuario@servidor:~$ cat /etc/shadow cat: /etc/shadow: Permissão negada root@servidor:~# cat /etc/shadow | grep "joao" joao:$6$xyz...hash...abc:19500:0:99999:7:::

👫 Gerenciando Grupos

Grupos são a forma mais eficiente de organizar permissões em servidores com múltiplos usuários. Em vez de configurar permissões individualmente, você cria um grupo com o nível de acesso desejado e adiciona os usuários relevantes.

bash — gerenciamento de grupos
# Criar um novo grupo root@servidor:~# groupadd desenvolvedores # Adicionar usuário a um grupo (use SEMPRE -aG, nunca apenas -G!) root@servidor:~# usermod -aG desenvolvedores maria root@servidor:~# usermod -aG desenvolvedores joao # Verificar membros do grupo root@servidor:~# getent group desenvolvedores desenvolvedores:x:1002:maria,joao # Remover grupo (só funciona se não for grupo primário de ninguém) root@servidor:~# groupdel desenvolvedores

Grupos importantes do sistema

Grupos de privilégio

  • sudo — membros podem usar sudo (Debian/Ubuntu)
  • wheel — equivalente a sudo no CentOS/RHEL
  • adm — pode ler logs do sistema em /var/log

Grupos de serviço

  • www-data — servidor web Apache/Nginx
  • docker — uso de containers Docker
  • dialout — acesso a portas seriais

⚠️ Diferença crítica: usermod -G grupo usuario substitui todos os grupos secundários existentes! Use sempre usermod -aG grupo usuario (com o -a de append) para apenas adicionar sem remover os grupos atuais.

🦸 sudo e o Arquivo /etc/sudoers

sudo significa Super User DO. Permite que um usuário comum execute comandos específicos com privilégios de root, sem precisar logar como root. É mais seguro porque registra tudo em log e exige autenticação do próprio usuário.

sudo comando

Executa um comando como root e volta. Usa sua senha. Registra no log.

⟷

su — (troca de usuário)

Abre um shell como root até você digitar exit. Usa a senha do root.

bash — sudo em ação
# sudo executa um comando como root e volta para usuário normal usuario@servidor:~$ sudo apt update [sudo] senha para usuario: Atingido:1 http://archive.ubuntu.com/ubuntu focal InRelease # Adicionar usuário ao grupo sudo (libera uso do sudo) root@servidor:~# usermod -aG sudo joao # Ver o que meu usuário pode fazer com sudo usuario@servidor:~$ sudo -l User usuario may run the following commands on servidor: (ALL : ALL) ALL # /etc/sudoers — arquivo de configuração (sempre edite com visudo!) root@servidor:~# visudo

Regra de ouro: Sempre edite o arquivo /etc/sudoers com o comando visudo, nunca diretamente com um editor de texto. O visudo valida a sintaxe antes de salvar — um erro no sudoers pode bloquear completamente o acesso administrativo ao sistema.

🔑 Senhas e Verificação de Usuários

Gerenciar senhas e verificar o estado dos usuários são tarefas rotineiras de um administrador. O comando passwd cuida de senhas, enquanto id, who, w e last fornecem informações sobre quem está no sistema.

Gerenciamento de senhas com passwd

bash — senhas e verificação
# Mudar minha própria senha usuario@servidor:~$ passwd Mudando senha para usuario. Senha atual: Nova senha: passwd: senha atualizada com sucesso # Root define senha de qualquer usuário root@servidor:~# passwd joao # Forçar troca de senha no próximo login root@servidor:~# passwd -e joao # Bloquear / desbloquear conta root@servidor:~# passwd -l joao # Bloqueia root@servidor:~# passwd -u joao # Desbloqueia

Verificando usuários no sistema

id joao Mostra UID, GID e todos os grupos do usuário
who Lista os usuários atualmente logados
w Logados + o que estão fazendo agora
last Histórico de logins e logouts do sistema
bash — monitorando usuários
usuario@servidor:~$ who usuario tty1 2024-01-15 10:30 maria pts/0 2024-01-15 11:45 usuario@servidor:~$ last -5 maria pts/0 Mon Jan 15 11:45 still logged in usuario tty1 Mon Jan 15 10:30 still logged in joao pts/1 Sun Jan 14 09:00 - 17:30 (08:30)

✏️ Modificando e Removendo Usuários

Com o tempo, contas precisam ser ajustadas: mudar de grupo, alterar o shell, bloquear temporariamente ou remover definitivamente. O usermod e userdel cobrem todas essas necessidades.

usermod — modificando contas existentes

bash — usermod e userdel
# Adicionar a grupo secundário (sempre use -aG!) root@servidor:~# usermod -aG sudo maria # Mudar o shell padrão root@servidor:~# usermod -s /bin/zsh joao # Bloquear conta temporariamente root@servidor:~# usermod -L joao # Desbloquear conta root@servidor:~# usermod -U joao # Remover usuário mantendo a pasta home (para backup) root@servidor:~# userdel joao # Remover usuário E sua pasta home (definitivo!) root@servidor:~# userdel -r joao

Opções do usermod

  • -aG grupo — adiciona ao grupo
  • -s shell — muda o shell
  • -d /novo/home -m — muda e move o home
  • -l novo_nome — renomeia o login
  • -L / -U — bloqueia / desbloqueia
  • -e 2025-12-31 — define expiração

Opções do userdel

  • sem flags — remove conta, mantém /home
  • -r — remove conta e a pasta home
  • -f — força remoção mesmo logado
  • deluser --remove-home joao — versão interativa (Debian)

🛠️ Atividade Prática — Administração de Usuários

⏱️ ~30 min 💻 Ubuntu Server / Terminal 📸 Tirar screenshot da saída
1
Criar o usuário aluno
Crie um novo usuário com pasta home usando: sudo useradd -m -s /bin/bash -c "Aluno de SO2" aluno. Confirme a criação consultando o arquivo de usuários: grep "aluno" /etc/passwd. Verifique se a pasta home foi criada: ls -la /home/aluno.
2
Definir a senha do usuário
Defina uma senha para o novo usuário: sudo passwd aluno. Digite uma senha segura quando solicitado. Em seguida, force a troca de senha no primeiro login: sudo passwd -e aluno. Isso é uma boa prática ao criar contas para terceiros.
3
Criar o grupo turma e adicionar o usuário
Crie um grupo chamado turma: sudo groupadd turma. Adicione o usuário aluno ao grupo (use -aG para não remover grupos existentes!): sudo usermod -aG turma aluno. Verifique se o grupo foi criado: getent group turma.
4
Adicionar ao grupo sudo e testar privilégios
Adicione o usuário aluno ao grupo sudo: sudo usermod -aG sudo aluno. Troque para o usuário aluno: su - aluno. Teste um comando administrativo: sudo ls /root. Se pedir senha e executar, o sudo está funcionando. Volte ao usuário original com exit.
5
Verificar com id e groups
Verifique todos os grupos do usuário aluno: id aluno. A saída deve mostrar UID, GID e os grupos: usuario, turma e sudo. Confirme também com groups aluno. Veja quem está logado no sistema agora com who e w.
6
Limpar: remover o usuário de teste
Para finalizar, remova o usuário aluno e sua pasta home: sudo userdel -r aluno. Confirme que o usuário foi removido: id aluno (deve retornar erro). Verifique que a pasta foi removida: ls /home/. Tire screenshot mostrando o resultado final de cada etapa.
📌 Para refletir: O gerenciamento de usuários é a base da segurança em Linux. Cada conta deve ter exatamente os privilégios necessários para sua função — nem mais, nem menos. Este princípio se chama mínimo privilégio e é fundamental em qualquer ambiente de produção. Um servidor comprometido por uma conta com excesso de permissões pode comprometer toda a infraestrutura.
