🧭 Navegação no Sistema de Arquivos

No Linux, tudo começa pelo terminal. Os três comandos de navegação mais usados são pwd, ls e cd. Dominá-los é o primeiro passo para trabalhar com eficiência.

pwd Mostra a pasta atual (Print Working Directory)
ls Lista arquivos e pastas do diretório
cd Navega entre diretórios (Change Directory)
find Busca arquivos e pastas no sistema
bash — navegação básica
usuario@servidor:~$ pwd /home/usuario usuario@servidor:~$ ls -la drwxr-xr-x 5 usuario usuario 4096 jan 15 Documentos/ drwxr-xr-x 2 usuario usuario 4096 jan 14 Downloads/ -rw-r--r-- 1 usuario usuario 2048 jan 15 notas.txt usuario@servidor:~$ cd Documentos usuario@servidor:~/Documentos$ # Agora estou em Documentos!

Variações úteis do ls

Opções de listagem

  • ls -l — lista detalhada com permissões, dono e tamanho
  • ls -a — mostra arquivos ocultos (que começam com .)
  • ls -la — detalhado + ocultos (combinação mais usada)
  • ls -lh — tamanhos em formato legível (KB, MB, GB)

Atalhos do cd

  • cd ~ ou cd — vai para a pasta home
  • cd .. — sobe um nível (pasta pai)
  • cd - — volta para a pasta anterior
  • cd / — vai para a raiz do sistema
  • cd ../Downloads — sobe e entra em outra pasta

Caminhos absolutos vs relativos: Caminhos que começam com / são absolutos (funcionam de qualquer lugar). Caminhos sem / são relativos à pasta atual. Exemplo: /home/usuario/Documentos (absoluto) vs Documentos/projetos (relativo).

📁 Manipulação de Arquivos e Pastas

Criar, copiar, mover e remover arquivos são operações diárias de um administrador. Os comandos abaixo cobrem praticamente todos os casos de manipulação de arquivos no terminal Linux.

Criar arquivos e pastas

  • mkdir pasta — cria um diretório
  • mkdir -p a/b/c — cria pastas aninhadas de uma vez
  • touch arquivo.txt — cria arquivo vazio ou atualiza data

Copiar e mover

  • cp origem destino — copia arquivo
  • cp -r pasta/ copia/ — copia diretório recursivamente
  • mv origem destino — move ou renomeia
bash — manipulação de arquivos
usuario@servidor:~$ mkdir -p projetos/site/css # Cria estrutura inteira usuario@servidor:~$ touch notas.txt # Cria arquivo vazio usuario@servidor:~$ cp notas.txt backup.txt # Copia arquivo usuario@servidor:~$ mv backup.txt /tmp/ # Move para /tmp usuario@servidor:~$ cp -r projetos/ backup_projetos/ # Copia pasta inteira

⚠️ rm não tem lixeira! O comando rm arquivo.txt remove permanentemente. Use rm -i arquivo para pedir confirmação antes de apagar. Para remover pastas com conteúdo, use rm -r pasta/ — mas com muito cuidado!

👀 Visualizando Conteúdo de Arquivos

Antes de editar ou processar um arquivo, é fundamental saber como visualizá-lo. O Linux oferece várias ferramentas, cada uma indicada para diferentes situações — de ver um arquivo pequeno inteiro até monitorar logs em tempo real.

cat Exibe o arquivo inteiro de uma vez
less Visualiza página por página (q para sair)
head Mostra as primeiras N linhas (padrão: 10)
tail Mostra as últimas N linhas do arquivo
bash — visualização de arquivos
usuario@servidor:~$ cat notas.txt Linha 1: Estudar Linux Linha 2: Praticar comandos usuario@servidor:~$ head -5 /var/log/syslog # Primeiras 5 linhas usuario@servidor:~$ tail -5 /var/log/syslog # Últimas 5 linhas usuario@servidor:~$ tail -f /var/log/syslog # Monitora em tempo real!

tail -f é uma das ferramentas mais valiosas para administradores: monitora um arquivo de log em tempo real, exibindo novas linhas assim que são escritas. Pressione Ctrl+C para sair.

🔍 Busca: grep, find e locate

Encontrar arquivos e conteúdo específico é uma habilidade essencial. O grep busca texto dentro de arquivos, enquanto find localiza arquivos pelo nome, tipo, tamanho ou data de modificação.

grep — busca padrões dentro de arquivos

bash — grep
usuario@servidor:~$ grep "erro" sistema.log # Busca "erro" no arquivo usuario@servidor:~$ grep -r "usuario" /etc/ # Busca recursivamente em /etc usuario@servidor:~$ grep -i "ERRO" sistema.log # Ignora maiúsculas/minúsculas usuario@servidor:~$ grep -n "falha" sistema.log # Mostra número da linha usuario@servidor:~$ grep -v "INFO" sistema.log # Mostra linhas que NÃO contêm

find — localiza arquivos no sistema

bash — find
usuario@servidor:~$ find /home -name "*.txt" # Busca arquivos .txt usuario@servidor:~$ find /var/log -mtime -7 # Modificados nos últimos 7 dias usuario@servidor:~$ find /tmp -type f -size +1M # Arquivos maiores que 1 MB usuario@servidor:~$ locate sshd.conf # Busca rápida via banco de dados

locate vs find: locate é muito mais rápido pois usa um banco de dados pré-indexado, mas pode estar desatualizado. find varre o disco em tempo real e suporta filtros avançados. Para atualizar o banco do locate, execute sudo updatedb.

🔗 Pipes e Redirecionamentos

Uma das maiores potências do Linux é a filosofia de compor comandos simples para resolver tarefas complexas. O pipe (|) encadeia a saída de um comando como entrada do próximo. O redirecionamento salva ou descarta saídas.

Pipe — encadeando comandos

comando1
|
comando2
|
comando3
bash — pipes
usuario@servidor:~$ ls -la | grep ".txt" # Lista e filtra .txt usuario@servidor:~$ ls | wc -l # Conta quantos arquivos há usuario@servidor:~$ ps aux | grep nginx # Localiza processo nginx usuario@servidor:~$ cat /var/log/syslog | grep "erro" | tail -20

Redirecionamentos — salvando e descartando saídas

Operadores de saída

  • > — redireciona saída, sobrescrevendo o arquivo
  • >> — redireciona saída, adicionando ao final
  • < — usa um arquivo como entrada de um comando
  • 2> — redireciona mensagens de erro (stderr)

Exemplos práticos

  • ls -la > lista.txt — salva listagem em arquivo
  • date >> log.txt — acrescenta data ao log
  • comando 2> erros.txt — salva só os erros
  • comando >> /dev/null 2>&1 — descarta tudo
bash — redirecionamentos
usuario@servidor:~$ ls -la > arquivos.txt # Salva listagem (sobrescreve) usuario@servidor:~$ echo "$(date)" >> log.txt # Acrescenta ao log usuario@servidor:~$ cat arquivo | grep "erro" > erros.log # Filtra e salva só os erros

⚡ Atalhos do Terminal e Produtividade

Conhecer os atalhos de teclado do terminal pode multiplicar sua velocidade no dia a dia. O Bash oferece uma série de combinações que evitam redigitar comandos e navegam no histórico rapidamente.

Tab — autocompletar

bash — Tab e histórico
usuario@servidor:~$ cd Doc[Tab] → cd Documentos/ # Autocompleta usuario@servidor:~$ cat rel[Tab] → cat relatorio.txt # Autocompleta arquivo usuario@servidor:~$ cd D[Tab][Tab] # Mostra todas as opções Desktop/ Documentos/ Downloads/ # Histórico de comandos usuario@servidor:~$ history 501 ls -la 502 cd Documentos 503 cat notas.txt usuario@servidor:~$ !502 # Re-executa comando 502 usuario@servidor:~$ !! # Re-executa o último comando

Atalhos essenciais de teclado

Controle de processo

  • Ctrl+C — cancela/interrompe o comando atual
  • Ctrl+Z — suspende o processo (para segundo plano)
  • Ctrl+D — encerra o terminal (EOF)
  • Ctrl+L — limpa a tela (equivale a clear)

Edição de linha

  • Ctrl+A — vai para o início da linha
  • Ctrl+E — vai para o fim da linha
  • Ctrl+U — apaga da posição até o início
  • Ctrl+K — apaga da posição até o fim
  • Ctrl+R — busca reversa no histórico

Dica de ouro: Ctrl+C é o seu "botão de emergência". Algo travou ou está demorando demais? Pressione Ctrl+C para interromper imediatamente. O Ctrl+R é excelente para resgatar comandos longos que você usou antes.

🚀 Combinando Comandos: Casos Reais

O verdadeiro poder do terminal aparece quando você combina comandos. Um administrador experiente consegue resolver em uma única linha o que levaria minutos com ferramentas gráficas.

bash — combinações práticas
# Encontrar e contar arquivos .log maiores que 10MB usuario@servidor:~$ find /var/log -name "*.log" -size +10M | wc -l # Mostrar as 10 últimas linhas de erro do syslog usuario@servidor:~$ grep -i "error" /var/log/syslog | tail -10 # Salvar lista de processos em arquivo de relatório usuario@servidor:~$ ps aux | grep -v "grep" >> processos.txt # Buscar arquivos de configuração que contêm "porta 22" usuario@servidor:~$ grep -r "Port 22" /etc/ 2> /dev/null # Listar os 5 maiores arquivos na pasta atual usuario@servidor:~$ ls -lhS | head -6

Prática recomendada: Quando um comando complexo der certo, salve-o em um arquivo de "receitas" pessoal. Com o tempo, você terá uma biblioteca de one-liners que resolve problemas recorrentes em segundos.

🛠️ Atividade Prática — Explorador do Terminal

⏱️ ~30 min 💻 Ubuntu Server / Terminal 📸 Tirar screenshot da saída
1
Navegar pelos diretórios principais
Abra o terminal e explore a estrutura: execute pwd para confirmar onde está, depois navegue com cd /var/log e liste os arquivos com ls -lh. Volte para a home com cd ~.
2
Criar uma estrutura de pastas de projeto
Crie a estrutura de um projeto com um único comando: mkdir -p ~/lab-terminal/{scripts,logs,backup}. Confirme a criação com ls -la ~/lab-terminal/. Crie três arquivos de texto dentro de scripts/ usando touch.
3
Copiar e mover arquivos
Copie um dos arquivos criados para a pasta backup/: cp ~/lab-terminal/scripts/arquivo1.txt ~/lab-terminal/backup/. Mova outro para logs/ e renomeie-o ao mesmo tempo: mv ~/lab-terminal/scripts/arquivo2.txt ~/lab-terminal/logs/registro.txt.
4
Buscar arquivos com find e grep
Use o find para localizar todos os arquivos .txt no seu diretório lab: find ~/lab-terminal -name "*.txt". Em seguida, adicione texto a um arquivo com echo "teste de busca com grep" > ~/lab-terminal/scripts/notas.txt e busque com grep -r "grep" ~/lab-terminal/.
5
Usar pipes para filtrar dados
Combine comandos com pipe: ls -la ~/lab-terminal/ | grep "^d" (mostra só pastas), ps aux | grep bash (mostra processos bash) e ls | wc -l (conta arquivos). Observe como a saída de um vira a entrada do próximo.
6
Redirecionar saída para arquivo de log
Gere um relatório do sistema redirecionando saídas: echo "=== Relatório $(date) ===" > ~/lab-terminal/logs/relatorio.txt, depois ls -la ~/lab-terminal/ >> ~/lab-terminal/logs/relatorio.txt. Visualize o resultado com cat ~/lab-terminal/logs/relatorio.txt e tire screenshot.
📌 Para refletir: O terminal Linux é uma linguagem de composição — cada comando pequeno resolve uma tarefa, e você os conecta com | e > para criar fluxos de trabalho poderosos. Um único administrador com bom domínio do terminal consegue gerenciar centenas de servidores com eficiência impossível de alcançar em ambientes gráficos.
