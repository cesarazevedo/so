📁 Sistema de Arquivos Linux

🌳 Estrutura Hierárquica

No Linux, tudo é um arquivo!
E tudo começa na raiz: /

/
├── bin/ - comandos básicos do sistema
├── boot/ - arquivos de inicialização
├── dev/ - dispositivos (hardware)
├── etc/ - configurações do sistema
├── home/ - pastas dos usuários
├── lib/ - bibliotecas compartilhadas
├── tmp/ - arquivos temporários
├── usr/ - programas dos usuários
└── var/ - dados variáveis (logs)

🏠 Diretórios Importantes

/home: suas pastas pessoais
/etc: configurações
/bin: programas essenciais
/usr: programas instalados
/var: logs e dados
/tmp: arquivos temporários

🔑 Permissões

rwxr-xr-x
│││││││││
│││└┴┴┴┴┴─ outros: r-x (ler+executar)
││└─ grupo: r-x (ler+executar)
└┴─ dono: rwx (ler+escrever+executar)
r: read (ler)
w: write (escrever)
x: execute (executar)

💡 Analogia

É como uma grande árvore genealógica,
onde cada pasta é um galho que pode ter mais galhos (subpastas)

🛠️ Criação e Remoção

✨ Criando e Removendo Arquivos e Pastas

Comandos para organizar seu sistema de arquivos

➕ Criação

📁 mkdir - Criar pastas

user@linux:~$ mkdir minha_pasta
user@linux:~$ mkdir -p pasta/subpasta
# -p cria pastas intermediárias

📄 touch - Criar arquivos

user@linux:~$ touch arquivo.txt
user@linux:~$ touch arquivo1.txt arquivo2.txt
# Criar múltiplos arquivos

🗑️ Remoção

🚮 rm - Remover arquivos

user@linux:~$ rm arquivo.txt
user@linux:~$ rm -f arquivo.txt # Força remoção
user@linux:~$ rm -r pasta/ # Remove pasta recursivamente
user@linux:~$ rm -rf pasta/ # Força + recursivo

📂 rmdir - Remover pastas vazias

user@linux:~$ rmdir pasta_vazia
# Só funciona se a pasta estiver vazia

⚠️ CUIDADO!

rm -rf é muito poderoso e perigoso! Pode deletar tudo sem avisar.
Sempre confira o caminho antes de executar. Não existe "lixeira" no terminal!

Slide 4 de 18

📋 Cópia e Movimentação

🔄 Copiando e Movendo Arquivos

Como duplicar, mover e renomear arquivos e pastas

📋 cp - Copiar

Arquivo original permanece

user@linux:~$ cp arquivo.txt copia.txt
user@linux:~$ cp arquivo.txt /tmp/
user@linux:~$ cp -r pasta1 pasta2
# -r para copiar pastas recursivamente

🚚 mv - Mover/Renomear

Arquivo original sai do lugar

user@linux:~$ mv velho.txt novo.txt
# Renomear arquivo
user@linux:~$ mv arquivo.txt /tmp/
# Mover para outra pasta
user@linux:~$ mv pasta1 pasta2
# Renomear ou mover pasta

😱 Diferença Crucial entre cp e mv

📋 cp (copiar):

Arquivo original permanece
Cria uma duplicata
Ocupa mais espaço

🚚 mv (mover):

Arquivo original sai do lugar
Vai para novo local
Mesmo espaço ocupado

📦 Gerenciamento de Pacotes

🛍️ Instalando e Removendo Software

Como instalar, atualizar e remover programas no Linux

📥 apt - Gerenciador Principal

user@linux:~$ sudo apt update
# Atualiza lista de pacotes
user@linux:~$ sudo apt upgrade
# Atualiza pacotes instalados
user@linux:~$ sudo apt install firefox
# Instala o Firefox
user@linux:~$ sudo apt remove firefox
# Remove o Firefox

🔍 Comandos Úteis

user@linux:~$ apt search firefox
# Busca pacotes relacionados
user@linux:~$ apt show firefox
# Mostra informações detalhadas
user@linux:~$ sudo apt autoremove
# Remove dependências órfãs

🔄 Fluxo Típico de Instalação

user@linux:~$ sudo apt update # 1. Atualizar lista
user@linux:~$ apt search nome-programa # 2. Buscar programa
user@linux:~$ sudo apt install nome-programa # 3. Instalar
user@linux:~$ nome-programa --version # 4. Verificar

⚠️ Sempre use sudo

Comandos de instalação/remoção precisam de privilégios administrativos.
sudo = "Super User DO" (execute como superusuário)

⚙️ Monitoramento de Processos

🔄 Gerenciando Processos em Execução

Como ver e controlar programas rodando no sistema

📋 ps - Processos Estáticos

user@linux:~$ ps
PID TTY TIME CMD 1234 pts/0 00:00:00 bash 5678 pts/0 00:00:00 ps
user@linux:~$ ps aux
# Todos os processos de todos os usuários

🔪 kill - Terminar Processos

user@linux:~$ kill 1234
# Termina processo com PID 1234
user@linux:~$ killall firefox
# Mata todos os processos Firefox

📊 top - Monitor em Tempo Real

user@linux:~$ top
Tasks: 195 total, 1 running, 194 sleeping %Cpu(s): 2.3 us, 1.1 sy, 96.6 id PID USER %CPU %MEM COMMAND 1234 user 5.3 2.1 firefox

⌨️ Navegando no top:

q sair • k matar processo • u filtrar por usuário

⚠️ Cuidado com kill -9

kill -9 força o término sem permitir que o programa se feche adequadamente.
Use primeiro kill normal, só use kill -9 se necessário.
