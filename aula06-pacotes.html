📦 Pacotes e Repositórios

Aula 6 — Instalação, atualização e remoção de software no Linux com apt e dpkg

📦 O que são Pacotes?

No Linux, um pacote é um arquivo compactado que contém tudo que um programa precisa para funcionar: os executáveis, arquivos de configuração, documentação e a lista de dependências (outros pacotes que ele precisa). No Debian e Ubuntu, esses arquivos têm extensão .deb.

🏪 Analogia: pense num pacote como um aplicativo da Play Store ou App Store — mas instalado pelo terminal. Você informa o nome, o sistema baixa, instala e configura tudo automaticamente.

O que há dentro de um pacote .deb

  • O programa compilado (binários executáveis)
  • Arquivos de configuração padrão
  • Documentação e páginas de manual (man)
  • Lista de dependências (outros pacotes necessários)
  • Scripts de pós-instalação (configuram o serviço automaticamente)

🏪 O que são Repositórios?

Um repositório é um servidor na internet que armazena milhares de pacotes organizados e verificados. O gerenciador apt consulta esses servidores para baixar e instalar softwares.

Tipos de repositórios no Ubuntu

  • main — Software oficial, mantido pela Canonical
  • universe — Software da comunidade, sem suporte oficial
  • restricted — Drivers proprietários (ex: NVIDIA)
  • multiverse — Software não-livre ou com restrições legais

Arquivo de configuração

A lista de repositórios configurados no sistema fica em:

bash
# Arquivo principal de repositórios /etc/apt/sources.list # Repositórios adicionais (PPAs e terceiros) /etc/apt/sources.list.d/

🔧 apt vs dpkg: qual usar?

Existem duas ferramentas principais para gerenciar pacotes no Debian/Ubuntu. Entender a diferença é essencial para escolher a correta em cada situação.

Característica apt dpkg
Nível Alto nível (fácil) Baixo nível
Dependências ✅ Resolve automaticamente ❌ Não resolve
Download ✅ Baixa da internet ❌ Precisa do arquivo .deb
Uso típico Dia a dia Instalar .deb baixado manualmente

💡 Regra prática: use apt para quase tudo. Use dpkg apenas quando você tiver um arquivo .deb local que não está nos repositórios (ex: Google Chrome, VS Code).

🔄 Atualizando o Sistema

Antes de instalar qualquer coisa, é boa prática atualizar a lista de pacotes disponíveis. São dois comandos distintos com funções diferentes.

bash
# 1. Atualiza a LISTA de pacotes disponíveis (não instala nada) root@servidor:~# apt update Atingido:1 http://br.archive.ubuntu.com/ubuntu jammy InRelease Obtido:2 http://security.ubuntu.com jammy-security InRelease Lendo lista de pacotes... Pronto # 2. Instala as atualizações dos pacotes já instalados root@servidor:~# apt upgrade Os seguintes pacotes serão atualizados: base-files bash coreutils... Deseja continuar? [S/n] S # Combo recomendado: atualiza a lista e já faz o upgrade root@servidor:~# apt update && apt -y upgrade

⚠️ update ≠ upgrade: apt update apenas atualiza o catálogo de pacotes disponíveis — não instala nada. É o apt upgrade que de fato instala as novas versões.

📥 Instalando Pacotes com apt install

O comando apt install busca o pacote nos repositórios, resolve as dependências automaticamente e instala tudo de uma vez.

bash
# Instalar um único pacote root@servidor:~# apt install htop Lendo listas de pacotes... Pronto Os seguintes NOVOS pacotes serão instalados: htop Deseja continuar? [S/n] S # Instalar vários pacotes de uma vez root@servidor:~# apt install vim git curl wget # Instalar sem pedir confirmação (útil em scripts) root@servidor:~# apt install -y nginx

💡 A flag -y responde “sim” automaticamente a todas as perguntas de confirmação. É muito útil em scripts de instalação automatizada, mas use com atenção: você não vai revisar o que será instalado.

🗑️ Removendo Pacotes

Há três comandos para remover software, cada um com um grau diferente de limpeza:

bash
# remove: desinstala o programa, mas MANTÉM as configurações root@servidor:~# apt remove htop # purge: remove o programa E os arquivos de configuração root@servidor:~# apt purge htop # autoremove: remove dependências que não são mais usadas root@servidor:~# apt autoremove Os seguintes pacotes serão REMOVIDOS: libx264-163 libx265-199...

Quando usar cada um?

  • remove — quando quiser reinstalar depois mantendo as configurações atuais
  • purge — quando quiser uma remoção completa, como se nunca tivesse instalado
  • autoremove — após remover pacotes, para limpar dependências que ficaram órfãs

🔍 Buscando e Consultando Pacotes

Antes de instalar, é possível pesquisar por nome ou descrição e ver informações detalhadas de um pacote.

bash
# Buscar pacotes por nome ou descrição usuario@servidor:~$ apt search editor vim - Vi IMproved - enhanced vi editor nano - small, friendly text editor gedit - official text editor of GNOME # Ver detalhes de um pacote específico usuario@servidor:~$ apt show htop Package: htop Version: 3.2.1-1 Description: interactive process viewer Depends: libc6, libncursesw6... # Listar pacotes instalados e atualizáveis usuario@servidor:~$ apt list --installed usuario@servidor:~$ apt list --upgradable

💾 dpkg: Instalando Arquivos .deb Locais

Quando um programa não está nos repositórios oficiais — como Google Chrome ou VS Code — você baixa o arquivo .deb manualmente e usa o dpkg para instalar.

bash
# Instalar um arquivo .deb baixado root@servidor:~# dpkg -i google-chrome-stable.deb # Se der erro de dependências, corrija com: root@servidor:~# apt install -f Corrigindo dependências... Pronto # Remover um pacote instalado via dpkg root@servidor:~# dpkg -r google-chrome-stable # Listar pacotes instalados e filtrar usuario@servidor:~$ dpkg -l | grep chrome

Principais flags do dpkg

  • -i — install (instalar um .deb)
  • -r — remove (remover, mantém configs)
  • -P — Purge (remove tudo, inclusive configs)
  • -l — list (listar pacotes instalados)
  • -S — Search (descobrir a qual pacote um arquivo pertence)

➕ Adicionando Repositórios Externos (PPA)

Quando um software não está nos repositórios oficiais, é possível adicionar um PPA (Personal Package Archive) — repositório mantido por terceiros no Launchpad da Canonical.

bash
# Adicionar um PPA root@servidor:~# add-apt-repository ppa:nome/ppa # SEMPRE atualize a lista após adicionar um PPA root@servidor:~# apt update # Agora pode instalar o pacote do PPA root@servidor:~# apt install nome-do-pacote # Remover um PPA root@servidor:~# add-apt-repository --remove ppa:nome/ppa

⚠️ Cuidado com PPAs! São mantidos por terceiros, sem garantia de qualidade ou segurança. Adicione apenas PPAs de fontes confiáveis e reconhecidas. Em servidores de produção, evite ao máximo o uso de PPAs.

🧹 Limpeza do Sistema

Com o tempo, o sistema acumula pacotes de cache e dependências que não são mais necessárias. Esses comandos liberam espaço em disco:

bash
# Remove dependências que não são mais usadas por nenhum pacote root@servidor:~# apt autoremove # Limpa TODOS os arquivos .deb do cache local root@servidor:~# apt clean # Limpa apenas .deb de versões antigas (mantém os atuais) root@servidor:~# apt autoclean # Combo recomendado de limpeza root@servidor:~# apt autoremove && apt autoclean

📌 Em servidores, é boa prática executar o combo de limpeza após grandes atualizações. Em ambientes com disco limitado, o cache de pacotes pode ocupar centenas de megabytes.

📋 Resumo: Comandos Essenciais

apt update Atualiza Lista Baixa info dos repos. Não instala nada.
apt upgrade Atualiza Pacotes Instala versões novas dos instalados.
apt install Instala apt install nome
apt remove Remove Mantém configurações.
apt purge Remove Tudo Inclui configurações.
apt search Busca Encontra pacotes por nome.
apt show Detalhes Versão, dependências, descrição.
dpkg -i Instala .deb Arquivo local sem repositório.
apt autoremove Limpa Órfãos Remove dependências inutilizadas.

⏰ Rotina recomendada de atualização: apt update && apt upgrade -y && apt autoremove

🛠️ Atividade Prática — Gerenciando Pacotes no Servidor

⏱ ~30 min 💻 Ubuntu Server / Terminal 📸 Tirar screenshot da saída
1
Atualize o sistema
Execute apt update && apt upgrade -y. Observe os repositórios sendo consultados e os pacotes atualizados. Tire screenshot da saída.
2
Instale e teste o htop
Execute apt install -y htop. Após instalado, rode htop para visualizar os processos em tempo real. Pressione q para sair.
3
Pesquise e inspecione um pacote
Busque editores de texto com apt search editor | head -10. Depois veja os detalhes do nano: apt show nano. Identifique a versão e as dependências.
4
Remova com remove e depois com purge
Remova o htop com apt remove htop. Em seguida use dpkg -l | grep htop para ver o status (deve aparecer como rc). Depois faça apt purge htop e verifique novamente.
5
Faça a limpeza do sistema
Execute apt autoremove && apt autoclean. Observe quantos pacotes e megabytes foram liberados. Tire screenshot mostrando o resultado.
📌 Para refletir: O gerenciador de pacotes é uma das maiores vantagens do Linux em relação ao Windows. Com um único comando, você instala, atualiza ou remove centenas de programas, com resolução automática de dependências e sem precisar acessar sites de terceiros.
