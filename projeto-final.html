🚀 PROJETO FINAL

Infraestrutura de Rede Completa com Serviços Integrados

3º Bimestre • Sistemas Operacionais de Redes

🎯 Objetivo do Projeto

Criar uma infraestrutura de rede completa e funcional integrando TODOS os serviços aprendidos no bimestre: DHCP, SAMBA, BIND9, Apache, SQUID e Webmin. O projeto simula um ambiente corporativo real onde os serviços trabalham juntos de forma integrada e organizada.

🔧 Serviços a Integrar

🌐 DHCP Server

Distribuição automática de IPs para os clientes da rede

📂 SAMBA

Compartilhamento de arquivos entre Windows e Linux

🔍 BIND9 (DNS)

Resolução de nomes de domínio personalizada

🌍 Apache

Servidor web hospedando sites locais

🔐 SQUID

Controle e monitoramento de acesso à internet

🖥️ Webmin

Interface de gerenciamento visual de tudo

📋 Requisitos Obrigatórios

Todos os 6 serviços instalados e funcionando
Serviços integrados (não isolados)
Configuração de rede adequada (IPs, DNS, etc)
Documentação clara de cada serviço
Testes comprovando funcionamento
Apresentação explicando a arquitetura

⚠️ CONFIGURAÇÕES OBRIGATÓRIAS - LEIA COM ATENÇÃO!

IP do Servidor: 192.168.10.X onde X = número do seu PC (ver etiqueta)
Domínio: NOME.SOBRENOME (use SEU nome! ex: joao.teixeira)
Exemplos:
• PC 5 → IP: 192.168.10.5 | Domínio: maria.silva
• PC 12 → IP: 192.168.10.12 | Domínio: pedro.santos
• PC 20 → IP: 192.168.10.20 | Domínio: ana.costa

🗺️ Topologia de Rede Sugerida

Servidor Ubuntu: 192.168.10.X/24 (IP estático)
⚠️ X = número do seu computador (ver etiqueta atrás do monitor ou no gabinete)
Hospeda DHCP, DNS, Apache, SQUID, SAMBA e Webmin
Clientes Windows: Recebem IP via DHCP (192.168.10.100-200)
Acessam compartilhamentos SAMBA, navegam pela internet via SQUID
Clientes Linux: Recebem IP via DHCP
Testam resolução DNS, acessam sites via Apache
Domínio Local: NOME.SOBRENOME
⚠️ USE SEU NOME! Exemplo: joao.teixeira, maria.silva, pedro.santos
Configurado no BIND9 para resolver nomes internos

🔗 Como os Serviços se Integram

1. Cliente liga na rede
⬇️
2. DHCP distribui IP automaticamente (192.168.10.100-200)
⬇️
3. DNS resolve nomes (www.NOME.SOBRENOME → 192.168.10.X)
⬇️
4. Apache serve sites acessíveis pelo nome
⬇️
5. SQUID controla acesso à internet
⬇️
6. SAMBA compartilha arquivos entre sistemas
⬇️
7. Webmin gerencia tudo via interface web

📦 O que Entregar

2. Documentação técnica (PDF ou Google Docs) explicando cada configuração
3. Prints de tela comprovando funcionamento de cada serviço
4. Diagrama de rede mostrando como tudo está conectado
