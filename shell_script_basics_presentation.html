🤔 Por que Aprender Shell Script?

🏠 Analogia da Casa

Imagine que você tem que fazer a mesma tarefa no computador todos os dias: criar uma pasta com a data de hoje, fazer backup de alguns arquivos e enviar um relatório por email. Fazer isso manualmente toda vez é chato e você pode esquecer passos, certo?

🤖 Shell Script = Robô Assistente

Automatiza tarefas repetitivas
Executa sempre da mesma forma
Não esquece nenhum passo
Funciona enquanto você faz outras coisas

⏰ Economia de Tempo

Exemplo: Se você gasta 10 minutos por dia numa tarefa repetitiva, isso são 60 horas por ano! Um script de 5 minutos pode economizar todo esse tempo.

👩‍🍳 Analogia da Cozinha

Se os comandos do Linux são como ingredientes soltos na cozinha, o shell script é como uma receita que combina esses ingredientes numa sequência específica para fazer um prato completo!

📈 Situações Práticas

Backup diário automático
Limpeza de arquivos temporários
Instalação de vários programas

📝 O que é um Shell Script?

📚 Definição Simples

Um shell script é simplesmente um arquivo de texto que contém uma lista de comandos que o Linux vai executar em sequência. É como uma lista de afazeres para o computador.

🧩 Componentes Básicos

Shebang (#!/bin/bash) - Como o "nome da receita" no topo
Comandos - Os ingredientes e passos da receita
Comentários - Suas anotações para lembrar o que faz

📜 Histórico Rápido

Nasceu com o Unix nos anos 1970
Programadores queriam automatizar tarefas
Nome "shell" = "casca" que protege o sistema

💡 Exemplo Simples

#!/bin/bash # Meu primeiro script - cria uma pasta e mostra a data echo "Olá! Vou criar uma pasta para você." mkdir minha_pasta echo "Pasta criada! Hoje é:" date echo "Pronto!"

🔍 Análise do Exemplo

#!/bin/bash - Diz qual interpretador usar
# comentário - Explicação para humanos
echo - Mostra mensagens na tela
mkdir, date - Comandos que você já conhece!

👨‍💻 Passo a Passo - Seu Primeiro Script

🎯 Vamos Criar: primeiro_script.sh

Um script que mostra informações básicas do sistema

📝 Passo 1: Criar o arquivo

user@linux:~$ vim primeiro_script.sh
# Isso abre o editor vim para criar o arquivo

✏️ Passo 2: Entrar no modo de inserção

Pressione: i
# Agora você pode digitar normalmente (modo INSERT)

📝 Passo 3: Escrever o conteúdo

#!/bin/bash
# Meu primeiro script
echo "Olá mundo do shell script!"
echo "Você está no diretório:"
pwd
echo "Hoje é:"
date

💾 Passo 4: Salvar e sair

Pressione Esc para sair do modo INSERT
Digite :wq e pressione Enter
:w = salvar (write)
:q = sair (quit)

🔑 Passo 5: Dar permissão

user@linux:~$ chmod +x primeiro_script.sh
# É como assinar uma receita dizendo "pode usar esta receita na cozinha"

🚀 Passo 6: Executar

user@linux:~$ ./primeiro_script.sh
# Importante: O ./ diz "procure aqui no diretório atual"

⚡ Comandos Essenciais do Vim

🎯 Regra de Ouro do Vim

Sempre pressione Esc primeiro se estiver em dúvida sobre qual modo você está!

📝 Para Escrever

i - Entra no modo INSERT (para digitar)
Esc - Sai do modo INSERT (volta ao normal)

💾 Para Salvar/Sair

:w - Salva o arquivo (write)
:q - Sai do vim (quit)
:wq - Salva e sai
:q! - Sai sem salvar (force quit)

🛠️ Comandos Úteis

dd - Deleta a linha inteira
u - Desfaz a última ação (undo)
yy - Copia a linha inteira
p - Cola o que foi copiado

🆘 Emergência: Travou no Vim?

Solução: Pressione Esc várias vezes, depois digite :q! e Enter

📋 Workflow Completo no Vim

vim nome_arquivo.sh → i → digite o código → Esc → :wq

🧠 Conceitos Fundamentais

🏷️ A) Shebang (#!/bin/bash)

O que é: Diz ao sistema qual "intérprete" usar
Analogia: Como escolher se vai cozinhar no fogão, forno ou microondas
Importante: Sempre na primeira linha, sem espaços antes

🔐 B) Permissões

Por que precisa: Linux é seguro - não executa qualquer arquivo
Como funciona: chmod +x = "pode executar"
Analogia: Como dar a chave da cozinha para alguém usar suas receitas

🚀 C) Execução

user@linux:~$ ./nome_do_script.sh
# ./ - Executa do diretório atual
# Por que o ./? - Diz "procure aqui no diretório atual"

❌ Erro Comum

nome_do_script.sh → "Command not found"
✅ Correto: ./nome_do_script.sh

💬 D) Comentários

# Isto é um comentário echo "Isto é código" # Comentário na linha
Começam com #
Para que servem: Explicar o que o código faz
Analogia: Como escrever na margem da receita "isso deixa mais saboroso"

💡 Boa Prática

Sempre use comentários - você vai esquecer o que fez depois!

🚨 Erros Comuns e Como Evitar

🔒 Erro 1: "Permission denied"

Problema: Esqueceu de dar permissão
Solução: chmod +x nome_do_script.sh

💡 Analogia

É como tentar dirigir sem carteira de motorista

❓ Erro 2: "Command not found"

Problema: Esqueceu do ./ antes do nome
Solução: ./nome_do_script.sh (não apenas nome_do_script.sh)

🛠️ Erro 3: "bad interpreter"

Problema: Shebang errado ou com espaços
Solução: Verificar se a primeira linha é exatamente #!/bin/bash

🚫 Erro 4: Travou no vim

Problema: Não consegue sair do vim
Solução: Pressione Esc várias vezes, depois :q! e Enter

🔧 Erro 5: Script não faz nada

Problema: Caracteres invisíveis ou codificação errada
Solução: Recriar o arquivo no próprio Linux

🔍 Dica de Debug

Use bash -x script.sh para ver linha por linha o que está acontecendo

⭐ Dicas de Boas Práticas

📝 1. Sempre use comentários

Você vai esquecer o que fez depois
Outros precisam entender seu código
# Este script faz backup dos arquivos importantes echo "Iniciando backup..."

🏷️ 2. Nomes descritivos

✅ backup_fotos.sh é melhor que script1.sh
✅ limpar_temp.sh é melhor que limpeza.sh

🧪 3. Teste sempre

Execute o script para ver se funciona
Teste com dados pequenos primeiro
Verifique se não quebra nada

🎯 4. Uma tarefa por vez

Comece simples, depois complique
Scripts pequenos são mais fáceis de debugar
Depois você pode combinar scripts

📁 5. Organize por pastas

Mantenha seus scripts organizados
Crie pastas por categoria
mkdir ~/scripts mkdir ~/scripts/backup mkdir ~/scripts/sistema

🔍 6. Lembrete do vim

Sempre pressione Esc antes de tentar sair
Evita confusão entre os modos

💡 Lembrete Final

Programar é como aprender a dirigir - no começo parece complicado, mas depois vira automático. A prática é fundamental!
