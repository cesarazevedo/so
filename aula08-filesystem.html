💾 Sistema de Arquivos e Montagem

Aula 8 — Filesystems, mount, umount, /etc/fstab e gerenciamento de dispositivos de armazenamento no Linux

🗂️ O que é Sistema de Arquivos?

O sistema de arquivos (filesystem) é a estrutura que define como os dados são organizados, armazenados e recuperados em um dispositivo de armazenamento. Cada partição ou disco precisa ter um filesystem para poder guardar arquivos.

📚 Analogia: Biblioteca. O disco é o prédio. O filesystem é o sistema de classificação (Dewey, alfabético). As pastas são as estantes. Os arquivos são os livros. Sem um sistema de classificação, os livros seriam pilões ilegíveis.

O que o filesystem define

  • Tamanho máximo de um arquivo individual
  • Tamanho máximo total da partição
  • Caracteres permitidos nos nomes de arquivos
  • Suporte a permissões e proprietários (chown, chmod)
  • Journaling: registro de operações para recuperação após falhas

⚠️ Importante: cada partição pode ter um filesystem diferente. Um mesmo disco pode ter /dev/sda1 em ext4 e /dev/sda2 em NTFS ao mesmo tempo.

📁 Tipos de Sistema de Arquivos

Existem diversos filesystems, cada um adequado para um cenário específico. Conhecer as diferenças ajuda a escolher o correto na hora de formatar.

🐧
ext4 Padrão Linux. Journaling, até 16 TB por arquivo. Use para discos internos.
🚀
XFS Alta performance. Ótimo para arquivos grandes e servidores de alta carga.
🆕
Btrfs Moderno. Suporta snapshots, compressão e subvolumes. Crescente adopção.
🪟
NTFS Padrão Windows. Use para partilhar dados com sistemas Windows.
📱
FAT32 Universal. Limite de 4 GB por arquivo. Pendrives simples e cartões SD.
🔄
exFAT FAT melhorado. Sem limite prático de tamanho. Pendrives e SSDs externos.

🔗 O que é Montagem?

No Linux, todo dispositivo de armazenamento existe como um arquivo em /dev/. Para acessar os dados contidos nele, é preciso montar o dispositivo: associar o dispositivo a uma pasta (chamada de ponto de montagem) na árvore de diretórios.

Nomenclatura dos dispositivos

  • /dev/sda — primeiro disco (SATA/SSD/USB)
  • /dev/sdb — segundo disco (ex: pendrive)
  • /dev/sda1 — primeira partição do primeiro disco
  • /dev/sdb2 — segunda partição do segundo disco

Pontos de montagem comuns

  • /mnt/ — montagem manual e temporária
  • /media/ — mídias removíveis (pen, HD externo)
  • /home — pode ser uma partição separada

💡 A pasta usada como ponto de montagem deve existir antes de montar. Crie-a com mkdir /mnt/pendrive se necessário.

🔧 Montando Dispositivos com mount

O fluxo para montar um dispositivo é sempre o mesmo: identificar, criar o ponto de montagem e montar.

bash
# Ver todos os dispositivos de bloco conectados root@servidor:~# lsblk NAME SIZE TYPE MOUNTPOINT sda 500G disk ├─sda1 499G part / └─sda2 1G part [SWAP] sdb 16G disk └─sdb1 16G part # ← pendrive, sem ponto de montagem # Criar o ponto de montagem root@servidor:~# mkdir -p /mnt/pendrive # Montar o dispositivo root@servidor:~# mount /dev/sdb1 /mnt/pendrive # Acessar os arquivos root@servidor:~# ls /mnt/pendrive fotos/ documentos/ backup.zip # Verificar espaço em disco (todos os pontos) root@servidor:~# df -h Filesystem Size Used Avail Use% Mounted on /dev/sda1 500G 120G 380G 24% / /dev/sdb1 16G 8G 8G 50% /mnt/pendrive

Opções úteis do mount

bash
# Montar como somente leitura root@servidor:~# mount -o ro /dev/sdb1 /mnt/pendrive # Especificar o tipo de filesystem (útil para NTFS) root@servidor:~# mount -t ntfs /dev/sdc1 /mnt/windows # Listar tudo que está montado no momento root@servidor:~# mount /dev/sda1 on / type ext4 (rw,relatime) /dev/sdb1 on /mnt/pendrive type vfat (rw)
  • ro — somente leitura (read-only)
  • rw — leitura e escrita (padrão)
  • noexec — impede execução de binários
  • nosuid — ignora bit SUID (segurança)
  • nofail — não aborta o boot se o dispositivo não for encontrado

⏏️ Desmontando com umount

Antes de remover fisicamente qualquer dispositivo (pendrive, HD externo), é obrigatório desmontá-lo. Remover sem desmontar pode corromper dados.

🚫 Nunca remova um pendrive ou HD externo sem antes executar umount. Dados em buffer ainda não gravados serão perdidos e o filesystem pode ser corrompido.

bash
# Desmontar pelo ponto de montagem root@servidor:~# umount /mnt/pendrive # Ou pelo dispositivo root@servidor:~# umount /dev/sdb1 # Erro: dispositivo ocupado? root@servidor:~# umount /mnt/pendrive umount: /mnt/pendrive: target is busy # Descobrir qual processo está usando o dispositivo root@servidor:~# lsof /mnt/pendrive COMMAND PID USER NAME bash 1234 joao /mnt/pendrive # Solução: sair da pasta e tentar novamente root@servidor:~# cd ~ root@servidor:~# umount /mnt/pendrive

💡 O erro “target is busy” acontece quando algum processo ou terminal está com o diretório montado como diretório atual. Basta executar cd ~ no terminal afetado e tentar desmontar novamente.

📋 /etc/fstab — Montagem Automática no Boot

O arquivo /etc/fstab lista todos os dispositivos que devem ser montados automaticamente quando o sistema inicia. É um dos arquivos mais importantes do Linux.

/etc/fstab
# <dispositivo> <ponto> <tipo> <opções> <dump> <pass> /dev/sda1 / ext4 defaults 0 1 /dev/sda2 none swap sw 0 0 /dev/sdb1 /dados ext4 defaults 0 2

Significado de cada coluna

1ª — Dispositivo /dev/sda1 ou UUID
2ª — Ponto /, /home, /dados
3ª — Tipo FS ext4, ntfs, vfat
4ª — Opções defaults, ro, nofail
5ª — Dump Backup? 0=não
6ª — Pass Ordem fsck: root=1, outros=2

⚠️ Erros no fstab podem impedir o boot! Sempre faça um backup antes de editar: cp /etc/fstab /etc/fstab.bak. Após editar, teste com mount -a antes de reiniciar.

🔑 UUID — Identificando Discos com Segurança

O nome de dispositivo /dev/sda1 pode mudar entre boots (dependendo da ordem de detecção do hardware). O UUID é um identificador único e permanente de cada partição — use-o no fstab para evitar problemas.

bash
# Descobrir o UUID de todos os dispositivos root@servidor:~# blkid /dev/sda1: UUID="a1b2c3d4-e5f6-7890-abcd-ef1234567890" TYPE="ext4" /dev/sdb1: UUID="1A2B-3C4D" TYPE="vfat" # Usar UUID no /etc/fstab (mais seguro que /dev/sdX) UUID=a1b2c3d4-e5f6-7890-abcd-ef1234567890 / ext4 defaults 0 1 UUID=1A2B-3C4D /mnt/usb vfat defaults,nofail 0 0 # Testar se o fstab está correto sem reiniciar root@servidor:~# mount -a

💡 Use a opção nofail no fstab para dispositivos removíveis (pendrives, HDs externos). Assim, se o dispositivo não estiver conectado, o sistema inicializa normalmente em vez de travar no boot.

🔧 Formatando Dispositivos com mkfs

Para preparar um novo disco ou mudar o filesystem de uma partição, usa-se a família de comandos mkfs (make filesystem).

🚨 CUIDADO: formatar apaga todos os dados da partição permanentemente e sem confirmação. Certifique-se de estar no dispositivo correto antes de executar.

bash
# Formatar como ext4 (padrão Linux) root@servidor:~# mkfs.ext4 /dev/sdb1 # Formatar como FAT32 (pendrives pequenos, compatível) root@servidor:~# mkfs.vfat -F 32 /dev/sdb1 # Formatar como exFAT (pendrives grandes) root@servidor:~# mkfs.exfat /dev/sdb1 # Formatar como NTFS (compatibilidade com Windows) root@servidor:~# mkfs.ntfs /dev/sdb1

📋 Resumo: Comandos Essenciais

lsblk Listar Discos Mostra dispositivos e partições.
mount Montar Conecta dispositivo a uma pasta.
umount Desmontar Desconecta antes de remover.
df -h Espaço em Disco Uso por ponto de montagem.
blkid Identificar UUID UUID e tipo de cada partição.
mkfs.ext4 Formatar mkfs.vfat, mkfs.ntfs, etc.

📌 Fluxo completo — montar um pendrive:
lsblk → mkdir /mnt/pendrive → mount /dev/sdb1 /mnt/pendrive → usar → umount /mnt/pendrive → remover com segurança.

🛠️ Atividade Prática — Montando e Gerenciando Discos

⏱ ~30 min 💻 Ubuntu Server / Terminal 📸 Tirar screenshot da saída
1
Inspecione os discos do sistema
Execute lsblk e depois df -h. Identifique: quantos discos existem, quais partições estão montadas e qual o espaço disponível em cada uma. Tire screenshot.
2
Leia o /etc/fstab e entenda cada linha
Execute cat /etc/fstab. Para cada linha ativa (sem #), identifique: o dispositivo ou UUID, o ponto de montagem, o tipo de filesystem e as opções. Execute também blkid para confirmar os UUIDs.
3
Adicione um disco virtual (loop device)
Crie um arquivo de 50 MB e o use como disco virtual:
dd if=/dev/zero of=/tmp/disco.img bs=1M count=50
mkfs.ext4 /tmp/disco.img
mkdir -p /mnt/teste
mount -o loop /tmp/disco.img /mnt/teste
df -h /mnt/teste Confirme que o disco virtual foi montado corretamente.
4
Crie arquivos e desmonte
Dentro do disco montado, crie alguns arquivos: echo "teste" > /mnt/teste/arquivo.txt && ls /mnt/teste. Depois desmonte: umount /mnt/teste. Verifique que o conteúdo não está mais acessível em /mnt/teste.
5
Remonte e verifique persistência
Monte novamente o mesmo arquivo: mount -o loop /tmp/disco.img /mnt/teste && ls /mnt/teste. Confirme que o arquivo criado no passo anterior ainda existe. Isso demonstra que o filesystem persiste no arquivo .img. Tire screenshot e desmonte ao finalizar.
📌 Para refletir: O conceito de montagem é central no Linux. Tudo é tratado como arquivo: discos, pendrives, partições e até imagens virtuais. O /etc/fstab é o que permite ao servidor inicializar e encontrar automaticamente todos os seus discos. Um erro nesse arquivo pode impedir o boot — por isso é tão importante entendê-lo bem.
