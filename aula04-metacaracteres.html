🔍 Metacaracteres e Expressões Regulares

Domine Padrões de Busca no Linux — De buscas simples a filtros poderosos!

📚 O que Vamos Aprender

Metacaracteres são símbolos especiais do shell que permitem criar padrões de busca poderosos.

exemplo inicial
usuario@linux:~$ ls arquivo*.txt
arquivo1.txt  arquivo2.txt  arquivoX.txt  arquivo_backup.txt
# O poder dos metacaracteres!

🔍 Metacaracteres do Shell (Wildcards)

  • Símbolo ? — substitui exatamente um caractere
  • Símbolo * — substitui zero ou mais caracteres
  • Colchetes [ ] — escolhe de um conjunto específico
  • Chaves { } — expansão em múltiplas opções

✅ Meta Final: Você vai conseguir fazer buscas inteligentes e poderosas como um verdadeiro ninja do terminal! 🥷

🕵️ Analogia: Metacaracteres são Lupas Especiais

Imagine que você é um detetive procurando pistas. Os metacaracteres são suas "lupas especiais" para encontrar padrões!

Símbolo Apelido Significado
? Lupa do "Exatamente Um" Vê 1 coisa qualquer, mas só 1
* Lupa do "Tudo ou Nada" Vê várias coisas ou nenhuma
[ ] Lupa Seletiva Vê só o que você escolher
{ } Lupa Multiplicadora Cria várias opções de uma vez
exemplo
detetive@linux:~$ ls pista?.txt
pista1.txt  pistaA.txt  pistaX.txt
# Encontrou 3 pistas! Cada uma com exatamente 1 caractere após "pista"

❓ Símbolo ? — A Lupa do "Exatamente Um"

O ponto de interrogação representa exatamente UM único caractere qualquer.

Característica Detalhe
Substitui Exatamente 1 caractere
Pode ser Letra, número ou símbolo
Não substitui "Nada" — precisa de exatamente 1

💡 Analogia: É como um coringa de baralho, mas que vale apenas uma carta! Ou uma vaga de estacionamento: cabe qualquer carro, mas só UM.

exemplos com ?
# Um ? = exatamente 1 caractere
usuario@linux:~$ ls test?.log
test1.log  test2.log  testA.log

# Dois ?? = exatamente 2 caracteres
usuario@linux:~$ ls doc??.pdf
doc01.pdf  docAB.pdf  docXY.pdf

# Múltiplos ? em posições diferentes
usuario@linux:~$ ls backup_202?-0?.tar
backup_2023-01.tar  backup_2024-05.tar  backup_2024-09.tar
Padrão Encontra Não encontra
file?.txt file1.txt, filea.txt, fileX.txt file.txt (zero chars), file12.txt (2 chars)

✅ O ? é perfeito quando você sabe exatamente quantos caracteres podem variar!

⭐ Símbolo * — A Lupa do "Zero ou Mais"

O asterisco representa zero ou mais caracteres — é o mais flexível dos metacaracteres.

💡 Analogia: É como um elástico mágico! Pode esticar para caber qualquer coisa, ou encolher para não ocupar nada.

exemplos com *
# Tudo que começa com "backup"
usuario@linux:~$ ls backup*
backup  backup.tar  backup_2024.zip  backup_completo.tar.gz

# Arquivos .conf que começam com "d"
usuario@linux:~$ ls /etc/d*.conf
dhcpd.conf  debug.conf  dns.conf

# Qualquer coisa que CONTENHA "log"
usuario@linux:~$ ls *log*
mylog.txt  syslog.conf  catalog.db  login.sh
Padrão Encontra Exemplo
*.txtTermina com .txta.txt, documento.txt
file*Começa com filefile, file1, file_backup
*log*Contém "log"mylog, syslog, catalog
a*zComeça com a, termina com zaz, abcz, amazing

⚖️ Comparação: ? vs *

? (Interrogação) * (Asterisco)
Exatamente 1 caractere Zero ou mais caracteres
Preciso: sabe quantos variam Flexível: não importa quantos
file?.txt → file1.txt file*.txt → file.txt, file1.txt, file12.txt...
? vs * na prática
# Temos: file.txt, file1.txt, file12.txt, fileABC.txt

usuario@linux:~$ ls file?.txt
file1.txt
# Só 1 caractere!

usuario@linux:~$ ls file*.txt
file.txt  file1.txt  file12.txt  fileABC.txt
# Todos!

💡 Quando usar cada um?
? → Quando você sabe EXATAMENTE quantos caracteres variam (ex: log01.txt até log99.txt)
* → Quando você NÃO sabe ou não importa quantos caracteres (ex: todos os .txt)

📋 Colchetes [ ] — A Lupa Seletiva

Os colchetes definem um conjunto específico de caracteres — escolhe 1 da lista, você decide quais são válidos.

💡 Analogia: É como um cardápio de restaurante! Você pode escolher qualquer prato do menu, mas só 1 de cada vez.

exemplos com [ ]
# Só aceita 1, 2 ou 3
usuario@linux:~$ ls file[123].txt
file1.txt  file2.txt  file3.txt

# Arquivos com números de 1 a 5
usuario@linux:~$ ls arquivo[1-5].txt
arquivo1.txt  arquivo3.txt  arquivo5.txt

# Arquivos que NÃO começam com 'a' ou 'b'
usuario@linux:~$ ls [^ab]*
config.txt  documento.pdf  foto.jpg
Sintaxe Significado
[a-z]Letras minúsculas
[A-Z]Letras maiúsculas
[0-9]Dígitos
[a-zA-Z]Qualquer letra
[0-9a-f]Hexadecimal
[^a]Qualquer coisa MENOS 'a'
[^0-9]Qualquer coisa que não seja número
exemplos avançados com [ ]
# Logs numerados de 00 a 99
usuario@linux:~$ ls log[0-9][0-9].txt
log01.txt  log15.txt  log42.txt  log99.txt

# Arquivos com letras A, B ou C
usuario@linux:~$ ls test[A-C]*.log
testA.log  testB_debug.log  testC_final.log

✅ Casos de uso: Logs numerados: log[0-9][0-9].txt — Versões: v[1-3].*.zip — Combine com *: [a-z]*[0-9].txt

🗂️ Chaves { } — A Lupa Multiplicadora

As chaves criam múltiplas variações de uma só vez — perfeito para criar vários arquivos ou expandir nomes.

💡 Analogia: É como uma máquina de cópias! Você dá um modelo e ela cria várias versões de uma vez.

exemplos básicos com { }
usuario@linux:~$ touch arquivo{1,2,3}.txt
# Cria: arquivo1.txt, arquivo2.txt, arquivo3.txt

# Criar arquivos de test1 até test10
usuario@linux:~$ touch test{1..10}.txt
# Cria: test1.txt test2.txt ... test10.txt

# Letras de A até E
usuario@linux:~$ touch versao_{A..E}.zip
# Cria: versao_A.zip versao_B.zip ... versao_E.zip

# Números com passo (2 em 2)
usuario@linux:~$ echo {0..10..2}
0 2 4 6 8 10
criando estrutura de pastas
# Criar estrutura completa de projeto
usuario@linux:~$ mkdir -p projeto/{src,docs,tests}
# Cria: projeto/src, projeto/docs, projeto/tests

# Múltiplos projetos com subpastas
usuario@linux:~$ mkdir -p empresa{1,2}/{vendas,ti,rh}
# Cria 6 pastas: empresa1/vendas, empresa1/ti...

# Backup rápido com { }
usuario@linux:~$ cp config.txt{,.backup}
# config.txt → config.txt.backup

# Renomear extensão
usuario@linux:~$ mv foto{.jpg,.jpeg}
# foto.jpg → foto.jpeg
combinando com outros metacaracteres
# Listar arquivos .conf OU .allow em /etc
usuario@linux:~$ ls /etc/*{conf,allow}
httpd.conf  hosts.allow  syslog.conf  tcp.allow

# Arquivos que começam com cron OU byobu
usuario@linux:~$ ls {cron,byobu}*
crontab  cron.daily  byobu.conf

⚖️ Diferença: [ ] vs { }

[ ] Colchetes { } Chaves
Filtra arquivos existentes Expande em múltiplos valores
Escolhe 1 caractere do conjunto Gera várias palavras
Só mostra se o arquivo existir Mostra todos, existindo ou não
[ ] vs { } na prática
# [ ] = FILTRA (só mostra se existir)
usuario@linux:~$ ls file[123].txt
file1.txt  file3.txt
# file2.txt não existe, não aparece

# { } = EXPANDE (gera os nomes)
usuario@linux:~$ echo file{1,2,3}.txt
file1.txt file2.txt file3.txt
# Mostra todos, existindo ou não

✅ Resumindo:
[ ] → Use para buscar/filtrar arquivos que já existem
{ } → Use para criar/expandir nomes (touch, mkdir, cp)

🚨 Erros Comuns e Como Evitar

⚠️ Erro 1: Confundir ? com *
file?.txt NÃO encontra file.txt
Solução: Use file*.txt para incluir "zero caracteres"

⚠️ Erro 2: Intervalos invertidos
[z-a] está ERRADO (ordem decrescente)
Solução: Use [a-z] (ordem crescente)

erro 3: deletar sem verificar
# ❌ PERIGOSO: Pode deletar arquivos importantes!
rm *.txt

# ✅ SEGURO: Sempre verifique primeiro!
usuario@linux:~$ ls *.txt
# Veja o que vai ser afetado
arquivo1.txt  importante.txt  teste.txt
usuario@linux:~$ rm *.txt
# Agora sim, delete

🛡️ Regra de Ouro: Sempre teste com ls antes de usar rm, mv ou cp!

🎯 Caso Prático: Organizando um Servidor

Você é o administrador de um servidor com milhares de arquivos de log. Vamos organizar!

organização do servidor
# 1. Ver todos os logs de 2024
admin@server:~$ ls log_2024*.txt
log_2024-01.txt  log_2024-02.txt  log_2024-03.txt ...

# 2. Ver logs apenas dos meses 01 a 06
admin@server:~$ ls log_2024-0[1-6].txt
log_2024-01.txt  log_2024-02.txt ... log_2024-06.txt

# 3. Criar backup de configurações
admin@server:~$ cp /etc/{apache2,nginx,mysql}/*.conf backup/
# Copia configs de apache2, nginx e mysql de uma vez!

# 4. Criar estrutura para novo projeto
admin@server:~$ mkdir -p projeto_web/{html,css,js,img}/{dev,prod}
# Cria 8 pastas organizadas!

✅ Com metacaracteres, tarefas que levariam horas são feitas em segundos!

❓ Verifique seu Conhecimento

O que representa o metacaractere * no shell Bash?

AQualquer caractere exatamente uma vez
BZero ou mais caracteres quaisquer (wildcard)
CApenas dígitos numéricos
DO início de uma linha
✓ * é o wildcard mais usado: ls *.txt lista todos os .txt. Em regex, significa “zero ou mais do caractere anterior”.

Qual metacaractere representa exatamente um único caractere qualquer no shell?

A*
B+
C?
D.
✓ O ? no shell substitui exatamente 1 caractere: ls arquivo?.txt encontra arquivo1.txt, arquivoA.txt, mas não arquivo10.txt.

O que faz o comando ls [abc]*.txt?

ALista arquivos .txt que começam com a, b ou c
BLista arquivos cujos nomes contêm [abc]
CLista arquivos .txt que terminam com a, b ou c
DLista arquivos com extensão abc.txt
✓ [abc] é uma classe de caracteres: combina um único caractere que seja a, b ou c. [a-z] combina qualquer letra minúscula.

Qual comando usa expressão regular para buscar linhas que começam com ‘root’ em /etc/passwd?

Afind /etc/passwd -name “^root”
Bls /etc/passwd | filter “^root”
Ccat /etc/passwd | match “^root”
Dgrep “^root” /etc/passwd
✓ O grep aceita regex: ^ ancora ao início da linha. grep -E ou egrep para expressões estendidas como +, ?, |.
