Monitoramento, Controle e Otimização — Domine o que roda no seu sistema!

🤔 O que é um Processo?

Processo = Programa em execução

Conceito Descrição
Programa Arquivo no disco (estático)
Processo Programa rodando na memória (dinâmico)

Analogia: Programa é a receita, processo é o bolo sendo feito!

Cada processo tem:

  • PID — Número único de identificação
  • PPID — PID do processo pai
  • Dono — Usuário que iniciou
  • Prioridade — Importância para a CPU
  • Estado — Rodando, dormindo, etc.
árvore de processos
# Todo processo tem um "pai" - exceto o primeiro (init/systemd)
usuario@linux:~$ pstree
systemd—└—apache2——5*[apache2]
        ├—sshd——sshd——bash——pstree
        └—mysql——mysqld

Imagine o Linux como uma fábrica com vários trabalhadores!

Fábrica (Sistema) Equivalente Linux
Mesas de trabalhoCPU
Espaço de trabalhoMemória RAM
TrabalhadoresProcessos
Crachá do funcionárioPID

Você é o gerente! Seus comandos:

  • ps — Ver lista de funcionários
  • top — Monitorar produtividade
  • kill — Demitir funcionário
  • nice — Definir prioridade
  • bg/fg — Mandar para 2º plano

🔄 Estados dos Processos

Estado Código Descrição
🏃 Running R Executando agora ou pronto para executar
😴 Sleeping S Esperando algo (entrada, tempo, etc.)
⏸️ Stopped T Pausado (Ctrl+Z ou sinal STOP)
🧟 Zombie Z Terminou, mas pai não coletou status

Novo → Pronto → Executando → Esperando → Terminado

💡 Processo Zombie? Não se preocupe! Zombies usam pouquíssimos recursos. São apenas "registros" de processos que terminaram. O sistema limpa automaticamente na maioria dos casos.

💀 Comando kill — Enviando Sinais

kill não é só "matar" — é enviar SINAIS para processos!

Sinais são mensagens que o sistema envia para processos. Analogia — é como um chefe mandando recados para funcionários:

  • "Termine seu trabalho" (SIGTERM)
  • "Pare AGORA!" (SIGKILL)
  • "Faça uma pausa" (SIGSTOP)
  • "Continue trabalhando" (SIGCONT)
sintaxe básica do kill
# Sintaxe geral
kill -SINAL PID

# Terminar processo 1234 (SIGTERM — padrão)
usuario@linux:~$ kill 1234

# Forçar término (SIGKILL)
usuario@linux:~$ kill -9 1234

# Ver lista de todos os sinais disponíveis
usuario@linux:~$ kill -l
 1) SIGHUP   2) SIGINT   3) SIGQUIT  9) SIGKILL
15) SIGTERM 18) SIGCONT 19) SIGSTOP 20) SIGTSTP
Número Nome Uso
1SIGHUPRecarregar configuração
2SIGINTInterromper (Ctrl+C)
9SIGKILLMatar forçado
15SIGTERMTerminar educadamente
18SIGCONTContinuar parado
19SIGSTOPPausar processo

⚠️ SIGKILL vs SIGTERM: Sempre tente SIGTERM (15) primeiro! SIGTERM permite que o processo salve dados, feche arquivos e libere recursos. SIGKILL (9) é último recurso — pode corromper dados!

exemplos práticos de kill
# 1. Terminar processo educadamente (SIGTERM - padrão)
usuario@linux:~$ kill 1234

# 2. Se não funcionou, forçar (SIGKILL)
usuario@linux:~$ kill -9 1234

# 3. Matar pelo nome (todos os processos com esse nome)
usuario@linux:~$ killall firefox

# 4. Matar por padrão de nome (regex)
usuario@linux:~$ pkill -f "python script.py"

# 5. Recarregar configuração do Apache (SIGHUP)
usuario@linux:~$ sudo kill -HUP $(pgrep apache2)
Variante Descrição
kill PIDPor PID específico
killall nomePor nome exato
pkill padrãoPor padrão/regex
xkillClique na janela (GUI)

🔄 Foreground e Background

Processos podem rodar na frente ou no fundo!

📺 Foreground (Frente) 🌙 Background (Fundo)
Ocupa o terminal Libera o terminal
Você vê a saída Roda "escondido"
Ctrl+C para parar Pode ter vários
Só 1 por terminal Use & no final
Ex: vim, nano, top Ex: downloads, backups
background com &
# Rodar em background (adicione &)
usuario@linux:~$ cp arquivo_grande.iso /backup/ &
[1] 5678
# Job 1, PID 5678

# Ver jobs em background
usuario@linux:~$ jobs
[1]+  Running  cp arquivo_grande.iso /backup/ &

⌨️ Controles: Ctrl+Z, bg, fg

ctrl+z, bg e fg na prática
# 1. Rodar um comando que demora...
usuario@linux:~$ find / -name "*.log" 2>/dev/null
(processando...)

# 2. Ops! Preciso do terminal. Pressione Ctrl+Z
^Z
[1]+  Stopped  find / -name "*.log"

# 3. Mandar para background e continuar
usuario@linux:~$ bg
[1]+ find / -name "*.log" &

# 4. Trazer de volta para foreground
usuario@linux:~$ fg
find / -name "*.log"
Comando Ação
Ctrl+ZPausar o processo atual
bgContinuar em segundo plano
fgTrazer para primeiro plano

🔒 nohup e disown — Processos Persistentes

E se eu fechar o terminal? O processo morre! nohup e disown resolvem isso.

nohup — No Hang Up
# Rodar script que sobrevive ao logout
nohup python backup.py &
nohup: saída anexada a 'nohup.out'
disown — remover da sessão
# Já iniciou? Use disown!
./script_longo.sh &
disown
# Agora pode fechar o terminal

💡 Quando usar:
nohup: Antes de iniciar um processo longo (backups, downloads)
disown: Esqueceu o nohup? Use disown depois de iniciar com &
screen/tmux: Para sessões complexas, considere usar screen ou tmux!

⚖️ nice e renice — Prioridades

Nem todo processo tem a mesma importância! nice = "gentileza" (quanto maior, menos prioridade)

Valor Nice Prioridade Comportamento
-20MáximaEgoísta — pega mais CPU
0NormalPadrão
+19MínimaGentil — deixa outros passarem
nice e renice
# Iniciar processo com prioridade baixa (gentil com outros)
usuario@linux:~$ nice -n 10 tar -czf backup.tar.gz /home

# Mudar prioridade de processo já rodando
usuario@linux:~$ renice -n 5 -p 1234
1234 (process ID) old priority 0, new priority 5

# Só root pode aumentar prioridade (nice negativo)
root@linux:~# nice -n -10 ./processo_importante

🔧 systemctl — Gerenciando Serviços

Serviços (daemons) são processos que rodam em background. Apache, MySQL, SSH, etc. são gerenciados pelo systemd.

gerenciando serviços com systemctl
# Ver status de um serviço
usuario@linux:~$ sudo systemctl status apache2
● apache2.service - The Apache HTTP Server
   Loaded: loaded (/lib/systemd/system/apache2.service)
   Active: active (running) since Mon 2024-01-15 10:00:00

# Iniciar / Parar / Reiniciar
root@linux:~# systemctl start apache2
root@linux:~# systemctl stop apache2
root@linux:~# systemctl restart apache2

# Habilitar/Desabilitar no boot
root@linux:~# systemctl enable apache2
root@linux:~# systemctl disable apache2

💡 Diferença: start vs enable
start/stop: Inicia/para AGORA
enable/disable: Inicia/não inicia no BOOT

🎯 Caso Prático: Servidor Lento

⚠️ 😱 O servidor está lento! Como investigar?

investigação passo a passo
# 1. Verificar carga do sistema
admin@server:~$ uptime
load average: 8.52, 7.21, 5.43
# Alto! Normal seria < 1

# 2. Ver quem está consumindo CPU/Memória
admin@server:~$ top -o %CPU
PID   %CPU %MEM  COMMAND
9876  95.2  45.1  python loop_infinito.py
# Vilão!

# 3. Investigar o processo
admin@server:~$ ps aux | grep 9876
joao  9876  95.2  45.1  python loop_infinito.py

# 4. Matar o processo problemático
admin@server:~$ kill 9876
# Se não funcionar: kill -9 9876

# 5. Verificar se melhorou
admin@server:~$ uptime
load average: 0.52, 2.21, 4.43
# Voltando ao normal!

❓ Verifique seu Conhecimento

O que é um processo em Linux?

AUm arquivo executável armazenado em disco
BUm programa em execução com PID, memória e recursos alocados
CUma tarefa agendada pelo cron
DUm serviço do systemd parado
✓ Processo = programa carregado em memória e em execução. Cada processo tem um PID único, processo pai (PPID), usuário dono e estado.

Qual comando envia o sinal SIGKILL (força o encerramento imediato) para o PID 1234?

Akill 1234
Bkill -15 1234
Ckill -9 1234
Dkillall -STOP 1234
✓ kill -9 envia SIGKILL — encerramento imediato, sem possibilidade de o processo se salvar. Use kill -15 (SIGTERM) primeiro para encerramento gracioso.

O que faz o comando jobs no terminal?

ALista os processos em segundo plano da sessão atual
BMostra todos os processos do sistema
CLista as tarefas do cron
DExibe o histórico de comandos executados
✓ jobs mostra processos suspensos (Ctrl+Z) e em background (&) da sessão atual. Use fg %1 para trazer o job 1 ao primeiro plano.

Como executar um comando em segundo plano no Bash?

Abackground comando
Brun -bg comando
Cnohup comando
Dcomando &
✓ Adicionar & ao final coloca o processo em background: sleep 100 &. O shell exibe o PID e retorna o prompt imediatamente.
