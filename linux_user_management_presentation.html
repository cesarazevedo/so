🏢 Analogia: Linux como uma Empresa

🤔 Vamos imaginar o Linux como uma empresa...

Isso vai facilitar muito o entendimento!

👤 Usuários = Funcionários

Cada pessoa tem um crachá único (UID)
Nome de usuário = nome no crachá
Pasta home = mesa/escritório pessoal
Senha = código para entrar na empresa
Shell = tipo de acesso ao sistema

👑 root = CEO da empresa

Pode fazer TUDO, contratar, demitir, reorganizar...
Mas é perigoso usar sempre. É como o CEO
fazer tarefas simples de funcionário!

👥 Grupos = Departamentos

TI, RH, Vendas, Marketing...
Cada departamento tem suas responsabilidades
Uma pessoa pode estar em vários departamentos
Facilita organizar e dar acessos

🔐 sudo = Chave mestra temporária

Como ter uma chave que abre qualquer porta,
mas só quando realmente precisar usar!
E sempre com registro de uso.

maria@empresa:~$ whoami
maria
maria@empresa:~$ groups
maria ti vendas
# Maria trabalha no TI e ajuda as Vendas
maria@empresa:~$ id
uid=1001(maria) gid=1001(maria) groups=1001(maria),1002(ti),1003(vendas)

👤 Criando Usuários

✨ Como "contratar" um novo funcionário

Vamos aprender a adicionar pessoas ao sistema

🆕 Comando useradd

root@linux:~# useradd joao
# Cria usuário básico
root@linux:~# useradd -m maria
# -m cria a pasta home
root@linux:~# useradd -m -s /bin/bash carlos
# -s define o shell padrão

🔑 Definir senha

root@linux:~# passwd maria
New password: ******** Retype password: ******** passwd: password updated successfully

🏗️ Exemplo Completo: Criando um usuário

root@linux:~# useradd -m -s /bin/bash ana
# 1. Cria usuária 'ana' com pasta home e shell bash
root@linux:~# passwd ana
New password: (digite uma senha forte)
Retype password: (repita a senha)
root@linux:~# id ana
uid=1001(ana) gid=1001(ana) groups=1001(ana)
# 2. Verifica se foi criada corretamente

💡 Dica: No Ubuntu/Debian, use adduser

root@linux:~# adduser pedro
# Comando mais amigável que pergunta tudo automaticamente
# Cria home, define shell, pede senha e informações extras

⚠️ Importante!

Sempre defina uma senha após criar um usuário!
Usuário sem senha = porta aberta para invasores.
Use passwd logo após useradd.

Slide 4 de 10

🔧 Modificando e Removendo Usuários

✏️ Modificar usuário (usermod)

root@linux:~# usermod -aG sudo maria
# Adiciona maria ao grupo sudo
root@linux:~# usermod -s /bin/zsh carlos
# Muda o shell do carlos
root@linux:~# usermod -L ana
# Bloqueia a conta da ana
root@linux:~# usermod -U ana
# Desbloqueia a conta
root@linux:~# usermod -c "Ana Silva" ana
# Define nome completo

🗑️ Remover usuário (userdel)

root@linux:~# userdel joao
# Remove usuário, mantém arquivos
root@linux:~# userdel -r joao
# Remove usuário E todos os arquivos

💀 PERIGO!

userdel -r apaga TUDO do usuário!
Sempre faça backup antes!

📋 Opções Úteis do usermod

Comando O que faz Exemplo
-aG Adiciona a um grupo usermod -aG developers ana
-s Muda o shell usermod -s /bin/zsh maria
-L / -U Bloquear / Desbloquear usermod -L baduser
-d Muda pasta home usermod -d /home/novo user
-c Define comentário/nome usermod -c "João Silva" joao

🏋️ Exercício Rápido

root@linux:~# useradd -m teste # 1. Criar usuário teste
root@linux:~# passwd teste # 2. Dar uma senha
root@linux:~# usermod -c "Usuario Teste" teste # 3. Definir nome
root@linux:~# id teste # 4. Verificar
root@linux:~# userdel -r teste # 5. Remover (cuidado!)

👥 Trabalhando com Grupos

🎯 Organizando por "departamentos"

Facilite sua vida agrupando usuários com funções similares

➕ Criar e gerenciar grupos

root@linux:~# groupadd developers
# Cria grupo "developers"
root@linux:~# groupadd designers root@linux:~# groupadd marketing root@linux:~# groupdel temp
# Remove grupo "temp"

👥 Gerenciar membros

root@linux:~# usermod -aG developers maria
# Maria entra no grupo developers
root@linux:~# gpasswd -a carlos marketing
# Outra forma de adicionar
root@linux:~# gpasswd -d joao developers
# Remove joao do grupo

🔍 Verificando grupos

user@linux:~$ groups maria
maria : maria sudo developers
user@linux:~$ getent group developers
developers:x:1001:maria,joao,ana
user@linux:~$ id maria
uid=1001(maria) gid=1001(maria) groups=1001(maria),27(sudo),1002(developers)
user@linux:~$ cat /etc/group | grep developers
developers:x:1001:maria,joao,ana

⚠️ Cuidado com -G vs -aG

usermod -G: Substitui TODOS os grupos (perigoso!)
usermod -aG: ADICIONA ao grupo (seguro)

root@linux:~# usermod -G developers maria # REMOVE dos outros grupos!
root@linux:~# usermod -aG developers maria # Adiciona, mantém outros

🏢 Exemplo Prático: Empresa de Software

root@linux:~# groupadd developers
root@linux:~# groupadd testers
root@linux:~# groupadd managers
root@linux:~# usermod -aG developers,testers maria
# Maria pode programar E testar
root@linux:~# usermod -aG managers,developers joao
# João é tech lead (gerencia e programa)

👑 O Poder do sudo

🔐 Super User DO: Poderes temporários

Como ter poderes de administrador sem os riscos do root

🤔 Por que sudo é melhor que root?

❌ Problemas do root

Poder demais = risco demais
Um comando errado quebra tudo
Difícil saber quem fez o quê
Sem controle de acesso

✅ Vantagens do sudo

Poder apenas quando necessário
Registra todas as ações
Cada um usa sua senha
Controle granular de permissões

⚡ Usando sudo

maria@linux:~$ sudo apt update
[sudo] password for maria: ********
maria@linux:~$ sudo useradd pedro
# Agora posso criar usuários
maria@linux:~$ sudo -i
# Viro root temporariamente
root@linux:~# exit
# Volto a ser maria

👥 Dando poder sudo

root@linux:~# usermod -aG sudo maria
# Maria ganha poderes sudo
root@linux:~# groups maria
maria : maria sudo
root@linux:~# visudo
# Editor seguro para configurar

⚠️ NUNCA edite /etc/sudoers diretamente!

Sempre use visudo que verifica se está certo.
Um erro pode quebrar o sudo inteiro e você perde acesso administrativo!

🛠️ Comandos úteis do sudo

user@linux:~$ sudo -l # Ver minhas permissões sudo
user@linux:~$ sudo -k # Limpar cache de senha
user@linux:~$ sudo tail /var/log/auth.log # Ver log de segurança
user@linux:~$ sudo su - outro_usuario # Trocar para outro usuário

💫 Lema do sudo

"Com grandes poderes vem grandes responsabilidades"
Use com sabedoria! 🕷️

🏋️ Exemplo Prático Completo

🎯 Cenário: Configurar uma Empresa

Vamos criar usuários e grupos para uma empresa de desenvolvimento

📋 O que precisamos fazer:

3 desenvolvedores: Ana, Bruno, Carlos
2 designers: Diana, Eduardo
1 gerente: Fernanda (com poderes sudo)
Organizar todos em seus respectivos grupos

1️⃣ Criando grupos

root@linux:~# groupadd developers
root@linux:~# groupadd designers
root@linux:~# groupadd managers

2️⃣ Criando usuários

root@linux:~# useradd -m -c "Ana Silva" ana
root@linux:~# useradd -m -c "Bruno Costa" bruno
root@linux:~# useradd -m -c "Carlos Santos" carlos
root@linux:~# useradd -m -c "Diana Lima" diana
root@linux:~# useradd -m -c "Eduardo Souza" eduardo
root@linux:~# useradd -m -c "Fernanda Alves" fernanda

3️⃣ Definindo senhas

root@linux:~# passwd ana
root@linux:~# passwd bruno
root@linux:~# passwd carlos
root@linux:~# passwd diana
root@linux:~# passwd eduardo
root@linux:~# passwd fernanda

4️⃣ Organizando grupos

root@linux:~# usermod -aG developers ana
root@linux:~# usermod -aG developers bruno
root@linux:~# usermod -aG developers carlos
root@linux:~# usermod -aG designers diana
root@linux:~# usermod -aG designers eduardo
root@linux:~# usermod -aG managers,sudo fernanda

✅ Testando se funcionou

root@linux:~# getent group developers
developers:x:1001:ana,bruno,carlos
root@linux:~# getent group designers
designers:x:1002:diana,eduardo
root@linux:~# groups fernanda
fernanda : fernanda managers sudo
root@linux:~# id ana
uid=1001(ana) gid=1001(ana) groups=1001(ana),1003(developers)

🎉 Resultado Final

Organização perfeita! Cada pessoa no seu grupo correto.
Desenvolvedores no grupo developers, designers no designers.
Fernanda pode administrar tudo com sudo e está no grupo managers.

💡 Dicas Finais

Ubuntu/Debian: Use adduser e deluser (mais amigáveis)
Sempre: Defina senhas após criar usuários

Backup: Sempre faça backup antes de deletar usuários
